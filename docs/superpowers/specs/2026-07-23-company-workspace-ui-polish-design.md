# 自有公司页精致工作台 UI 升级设计

## 1. 目标

在**不改业务功能与路由**的前提下，把「自有公司」总览页与详情工作台做成：**精致、有图形感、不古板**。

用户确认方向：

- 方案 A：精致工作台升级
- 商务感要有，但不能土、不能花、不能大屏运营风
- 需要适度图形化（比例条、状态徽章、轻渐变），而不是纯文字堆叠

## 2. 范围

### 2.1 包含

- `Pages/Companies/Index.cshtml` 视觉与排版
- `Pages/Companies/Details.cshtml` 标题区、速览条、标签、各 tab 内容壳层
- `_CompanyScopeSwitcher.cshtml` 下拉外观
- `wwwroot/css/pages.css` 中公司工作台专用样式
- 少量 `components.css` 仅当复用组件缺陷时做兼容（优先 pages 作用域，避免波及全站）

### 2.2 不包含

- 改 tab 业务逻辑、服务查询、权限
- 重做全站主题/侧栏
- 引入第三方图表库
- 新增复杂 JS 动画框架

## 3. 设计原则

1. **统一**：沿用现有 token（`--app-primary`、圆角、阴影、玻璃感）。
2. **层次**：标题 → 指标/速览 → 主内容；主操作不超过一排。
3. **图形化克制**：用比例条、色点、徽章、轻渐变；禁止大面积刺眼色块。
4. **密度适中**：比现状更紧凑，但不挤；桌面优先 1440，兼容 1280。
5. **状态可读**：启用/停用、临期/过期、未收款高低，一眼可辨。

## 4. 全部公司页 `/Companies`

### 4.1 页头

- 左侧：眉标 `公司维度经营统计` + 标题 `自有公司` + 一行浅灰说明
- 右侧：公司范围下拉 +「新增公司」
- 页头改为横向 flex，垂直居中，间距 `1rem`
- 下拉做成“胶囊选择器”：浅底、细边、聚焦主色描边

### 4.2 指标卡（2 张）

保持字段：公司数量、未收款。

视觉升级：

- 两卡等宽网格（桌面 `1fr 1fr`，窄屏单列）
- 顶部 3px 主色渐变条（沿用 metric-card）
- 数字更大：`clamp(1.6rem, 2.4vw, 2.1rem)`
- 未收款卡：
  - 副文案显示 `已收 / 应收` 小字
  - 底部增加**收款完成比例条**（已收/应收，分母 0 则 0%）
- 公司数量卡：
  - 副文案 `权限范围内可见公司`
  - 可选轻量装饰点，不用大图标库

### 4.3 金额对比面板

把现有 4 行“标签 + 金额”升级为**标签 + 金额 + 双段/单段比例条**：

| 行 | 左值 | 右值 | 条含义 |
| --- | --- | --- | --- |
| 应收 / 已收 | 应收 | 已收 | 已收占应收 |
| 应付 / 已付 | 应付 | 已付 | 已付占应付 |
| 销项 / 进项 | 销项 | 进项 | 进项相对销项（或并列双条，实现选更清晰者） |
| 工资 / 设备 | 工资 | 设备 | 并列双条或占比条 |

实现偏好：

- 优先复用 `.progress-track` / `.progress-fill`
- 行距收紧到约 `0.75–0.9rem`
- 金额右对齐、等宽数字

### 4.4 公司列表

- 面板标题区保留「公司档案」
- 搜索工具条与表格拉近，减少空洞
- 公司名链接加粗、主色
- 状态改为 pill：启用绿 / 停用灰
- 操作列：`查看 / 编辑 / 复制 / 证书` 改为轻量文字按钮间距，不再用生硬的 ` · ` 堆叠（可用 flex gap）
- 表头背景与项目列表一致，hover 行浅底

## 5. 具体公司页 `/Companies/Details/{id}`

### 5.1 页头

- 主标题改为**公司全称**（不是“自有公司”重复）
- 眉标：`编码 · 分类`
- 副行：简称 + 状态徽章（启用/停用）
- 右侧：范围下拉 + 复制/新增 + 返回全部公司
- 返回按钮用 secondary，主操作 primary 仅保留必要时

### 5.2 经营速览条（4 格）

字段不变：应收/已收、应付/已付、未收款、未付款。

视觉：

- 四列等宽圆角卡，浅表面 + 极轻阴影
- 每格上标签小字，下主数字
- 未收款/未付款：数值偏强调色（warning/danger 仅当 >0）
- 可选每格左侧 3px 色条区分四类，避免“四张白板”

### 5.3 标签导航

- 容器：横向滚动，底部分割线
- 项：中文标签，默认 muted
- 当前项：主色字 + 底部 2px 指示条（或柔和胶囊，二选一；推荐**底线指示**更精致、不土）
- hover：文字变深，不跳动布局

### 5.4 各 Tab 内容壳

统一：

- 主内容区 `display:grid; gap: 1rem`
- panel 内边距桌面 `1.1–1.25rem`
- 空状态居中浅文案 + 可选虚线框

#### 经营概览
- 左：金额对比（同 4.3 图形条）
- 右：结构摘要改为 2×N 统计瓷片（项目/合同/账户/证书）
- 下方近期动态表：类型列用小 pill（项目/收款/付款…）

#### 基本信息
- 保持 inline-edit
- 字段网格更整齐（4 列→窄屏 2/1）
- 标题操作按钮右对齐、尺寸统一 small/action

#### 证书
- 到期状态徽章颜色体系与全站 pill 对齐
- 表空状态更友好

#### 账户
- 停用行：灰底 + `[已停用]` 徽章 + 状态 pill
- 启停按钮：ghost/small，确认文案已定不改
- 新增账户表单：两列网格，按钮区 sticky 或右下对齐

#### 项目与合同 / 收付款与发票
- 只读表强化：链接色统一、金额右对齐
- 区块标题下加一行 muted 说明（已存在可润色）
- “查看全部”按钮与标题同行

## 6. 公司范围下拉

- class 固定：`company-scope-switcher` / `company-scope-select`
- 高度与 primary 按钮对齐（约 2.55rem）
- 标签“公司范围”可改为更短或图标+文字（实现时保可访问性 label）
- 选中项文字截断省略，避免把页头撑破

## 7. 响应式

- `>=1200`：指标两卡、对比+列表/摘要两列
- `760–1199`：速览 2×2，标签可横滑
- `<760`：单列，按钮换行，表格横滑

## 8. 实现策略

1. **优先 CSS 作用域**  
   使用 `.company-workspace` / `[data-company-dashboard]` 包裹，避免污染其他模块。
2. **HTML 只加 class 与结构钩子**  
   为比例条补充宽度 style/CSS 变量；不改 handler。
3. **计算**  
   比例在 Razor 内简单计算：`received/receivable`，clamp 0–100。
4. **验收**  
   桌面截图：Index、overview、profile、accounts 四张对比前后；保持原有功能测试通过。

## 9. 明确不做

- 不换字体家族
- 不加拟物重阴影
- 不做 3D/粒子/大插画
- 不在本轮改手机端专属视觉语言

## 10. 成功标准

- 用户主观：明显更精致、有图形、不土
- 信息层级一眼可扫：先数字后表格
- 功能回归：tab、启停、编辑、跳转仍可用
- 自动化：现有 Company 相关测试仍通过（断言若依赖旧 class 文案则同步微调）
