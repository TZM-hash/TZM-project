# 项目总览、统一工程量与明细附件设计

## 目标

将财务年度迁入系统设置；压缩项目总览；把工程量从暂估/结算双套字段简化为单套数据；以项目阶段解释暂估或结算；为工程量、收款、开票、付款和施工记录提供逐条多附件能力，并保证各标签的详细编辑只管理对应业务。

## 已确认规则

- 所有公司、外部账本和内部账本共用同一套财务年度，员工业务年度继续独立。
- 项目阶段新增“部分结算”，项目阶段是金额口径的唯一来源；部分结算和已结算按结算金额解释，其他阶段按暂估金额解释。
- 工程量明细只保存一套工程量和单价，小计统一为工程量乘单价。
- 工程量“口径”仅为人工备注，不参与计算。
- “是否开票”由人员手动选择，默认是；总应开票金额为勾选明细的含税小计之和。
- 每条工程量、收款、开票、付款和施工记录可关联多个附件。
- 五个标签的详细编辑只管理对应业务，不修改项目总览。
- 项目总览卡片高度压缩到当前约三分之二；税金与施工机械同一行各占一半；联系人和电话合并显示；工程金额使用暂估/结算下拉显示。
- 最终不做手机端验收。

## 数据迁移

- 暂估口径项目优先保留原暂估数量和单价。
- 部分结算或已结算项目优先保留原结算数量和单价。
- 首选值为空时回退另一套已有值。
- 保持工程量明细主键不变，确保财务分摊和工作簿关联连续。
- 历史工程量的“是否开票”初始化为是。

## 汇总规则

- 总工程款 = 全部工程量明细小计之和。
- 总应开票金额 = 需要开票的工程量明细小计之和。
- 已开票金额继续来自有效开票记录。
- 未开票金额 = 总应开票金额 - 已开票金额。
- 税率不额外叠加到工程量小计；工程量金额已经含税。

## 验证

- 单元测试覆盖项目阶段、工程量小计、工程款和应开票金额。
- 迁移测试覆盖新字段、历史数据选择和主键保留。
- Web 测试覆盖财务年度入口、紧凑布局、标签独立详细编辑和附件入口。
- 应用测试覆盖五类附件的权限、上传、下载和删除。
- Release 构建、定向测试、全量测试和桌面浏览器验收通过；不执行手机端验收。
