# 项目清单紧凑导出设计

## 目标

移除项目管理页的“导出当前视图”总览导出，以及表格上方占据整行的项目工作簿导出面板。把项目工作簿导出能力移动到数据工具栏右侧，以一个紧凑的“导出项目清单”下拉入口呈现。

## 已确认方案

- 工具栏右侧使用与“列管理”一致的原生 `details` 下拉交互，入口文案为“导出项目清单”。
- 下拉菜单右对齐，桌面端宽度约 22rem；移动端限制在视口内。
- 菜单分为导出范围、工作簿细项、附件和操作区。
- 当前筛选全部与表格行勾选继续互斥；已勾选项目优先。
- 保留 `ExportWorkbook` 后端处理、权限判断、附件 ZIP 和工作簿细项选择。
- 移除项目总览 `Export` 表单、处理方法和相关服务依赖，其他页面的共享工作台导出不受影响。

## 视觉与交互

- 沿用现有蓝色主按钮、8px 圆角、边框、阴影和工具栏字号，不引入新的视觉体系。
- 菜单标题展示“项目清单导出”，并用动态文本显示当前已勾选项目数量。
- 导出范围使用清晰的复选项；工作簿细项使用紧凑两列网格。
- 最终“生成项目工作簿”按钮占满菜单宽度，避免在窄工具栏中堆叠多个操作按钮。
- 支持 Esc 关闭及点击菜单外关闭。

## 验收标准

- 项目管理页不再显示“导出当前视图”。
- 表格上方不再显示独立导出面板。
- 有权限角色在工具栏右侧看到“导出项目清单”，无权限角色看不到。
- 勾选项目、导出全部筛选结果、选择工作簿细项和附件 ZIP 均可正常提交原 `ExportWorkbook` 处理方法。
- 桌面和移动端无横向溢出、遮挡或不可读文本。
