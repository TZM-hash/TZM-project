# 全业务 Excel 数据交换设计

**日期：** 2026-07-19  
**状态：** 已完成业务讨论，待用户审阅书面规格

## 1. 目标

在现有导入导出能力上建立覆盖全部用户可维护业务字段的 Excel 数据交换体系，使用户能够：

- 从各模块按当前筛选和字段选择导出 Excel。
- 选择一个、多个或全部大模块导出。
- 选择单工作簿多工作表，或多个模块工作簿打包 ZIP。
- 在 Excel 中通过快捷链接访问关联工作表、关联记录和附件。
- 下载空白模板后批量新增数据。
- 导出系统数据、在 Excel 中修改后重新导入更新原记录。
- 上传任意来源 Excel，通过字段映射导入旧系统或自定义表格。
- 在权限、脱敏、关联、并发和业务校验不被绕过的前提下完成整批导入。

本设计将现有 `/DataExchange` 从表单集合改造成统一数据交换中心，同时保留各模块现有列表页上的日常导出入口。

## 2. 当前现状与问题

当前系统已经具备：

- 部分数据集的字段目录、字段选择和 Excel 导出。
- 用户上次导出字段选择、个人模板和管理员共享模板。
- 部分主档、证书和设备数据的标准模板、预览和确认导入。
- 导入批次、原始文件、错误明细和列名映射的基础模型。
- 项目和财务列表的服务端筛选、排序、分页、列管理和当前视图导出。

主要缺口是：

- 数据集枚举、真正可导出数据集和可导入数据集不一致。
- 多数模板只覆盖部分字段，合同、工程量、完整工资、关联关系等缺少完整双向交换。
- 当前页面把字段目录、导出、模板保存和导入堆叠在同一页面，层级不清晰。
- 导出页面没有清楚区分全部数据、当前筛选、字段选择和多模块导出。
- 标准模板、导出后更新和任意 Excel 映射没有形成统一向导。
- 缺少多模块工作簿、导航目录、跨表链接、附件 ZIP 和完整任务历史。

## 3. 范围边界

### 3.1 Excel 覆盖范围

Excel 覆盖全部用户可维护的业务数据和基础配置字段，包括：

- 组织、部门、分支机构、自有公司和公司基础配置。
- 项目、合同、工程量、项目签约公司、项目税金、项目人员、合作单位和里程碑。
- 财务结算、发票、收付款、账户、抵扣和关联分配。
- 员工、人员归属、员工证书、工资应付、领款、报销、借支、分红及其他往来。
- 工资批次、工资项目、工资发放、班组、班组人员和工资代发关联。
- 合作单位、业务角色和联系人。
- 设备档案、权属、租赁、项目使用、日期段、维保和结算。
- 公司证书、阶段成果、备注和附件元数据。
- 业务年度、分类字典和用户可维护的系统设置。

### 3.2 普通 Excel 不覆盖的内部数据

以下内容不进入普通业务 Excel：

- 密码哈希、登录令牌和服务器密钥。
- 审计日志、提醒记录、备份任务和恢复任务。
- EF 迁移记录、迁移映射和其他技术内部表。
- 浏览器离线草稿和本机缓存。
- 服务器连接字符串、TLS 证书和私钥。

这些内容由完整系统备份负责。

### 3.3 非目标

- 不把 Excel 当作系统备份格式。
- 不允许 Excel 绕过业务服务直接写数据库。
- 不允许通过 Excel 物理删除业务记录。
- 不自动暴露所有数据库列。
- 不为即将被统一财务账本替换的旧财务模型建立长期模板。

## 4. 总体架构

采用“模块入口 + 统一数据交换中心 + 显式数据集目录”的方案。

### 4.1 模块入口

各业务列表页继续使用现有筛选、排序、列管理和个人视图，并提供：

- 导出当前视图。
- 自定义字段导出。
- 当前模块全量导出。

页面不实现独立 Excel 生成逻辑，而是把页面筛选状态和字段状态传给统一导出服务。

### 4.2 统一数据交换中心

`/DataExchange` 负责：

- 多模块或全模块导出。
- 单工作簿和 ZIP 格式选择。
- 包含附件选项。
- 空白模板下载。
- 系统导出文件更新导入。
- 任意 Excel 字段映射。
- 导入预览、确认、错误报告和任务历史。

### 4.3 显式数据集目录

每个数据集显式定义：

- 数据集键、中文名称、所属大模块和工作表名。
- 字段键、中文名称、数据类型和字段顺序。
- 是否必填、是否可导入、是否只读计算字段。
- 新增键、更新键和关联业务键。
- 权限、脱敏和敏感字段要求。
- 依赖数据集和导入顺序。
- 枚举值、清空规则、最大长度和业务校验器。
- 附件关联和超链接目标。

运行时只执行目录中明确登记的字段。数据库模型反射仅用于测试目录覆盖率，不用于自动暴露字段。

## 5. 数据模块与唯一归属

大模块分为：

1. 组织、自有公司与系统基础配置。
2. 项目、合同、工程量与项目关联。
3. 财务、账户、发票、收付款及关联分配。
4. 员工、年度总账、工资与人员归属。
5. 施工班组、班组人员与工资代发。
6. 合作单位、角色与联系人。
7. 设备、租赁、使用、日期段、维保与结算。
8. 员工/公司证书、阶段成果与附件。
9. 业务年度、分类字典和用户可维护设置。

每条业务记录只属于一个主数据集。其他工作表通过稳定系统 ID 和可读业务编号引用，不保存可独立编辑的副本。

例如：

- 员工只在员工主表保存一次，工资表引用员工 ID 和员工编号。
- 项目只在项目主表保存一次，合同和财务表引用项目 ID 和项目编号。
- 发票只在最终财务发票数据集保存一次，公司、项目和合作单位工作表通过关联字段读取。

## 6. 字段类型与编辑规则

字段分为四类：

### 6.1 可编辑业务字段

支持导出和重新导入。空白模板用于新增；系统导出文件用于更新。

### 6.2 关联字段

显示可读业务编号和名称，同时保留隐藏系统 ID。导入时优先使用系统 ID，新增模板使用业务编号解析关联。

### 6.3 计算字段

余额、进度、剩余金额、汇总金额和风险标记等字段只允许导出，不允许导入覆盖。

### 6.4 控制字段

包括系统 ID、并发版本、数据集版本和导入操作类型。默认隐藏并保护，用户不得删除或修改其格式。

## 7. 导出设计

### 7.1 导出方式

- **导出当前视图：** 当前筛选结果和当前字段顺序。
- **自定义导出：** 当前筛选结果，重新选择字段和顺序。
- **全量导出：** 当前权限范围内全部记录和全部可导出字段。
- **多模块导出：** 在数据交换中心选择一个、多个或全部模块。
- **模板导出：** 使用个人模板或管理员共享模板。

### 7.2 筛选与最后选择

不同页面可以拥有不同筛选项，但共用现有数据工作台底层。按“用户 + 页面 + 数据集”保存：

- 查询条件。
- 排序方式。
- 显示字段和字段顺序。
- 行密度和分页大小。
- 最后一次导出范围、文件结构和附件选项。

再次进入页面自动恢复最后选择，并提供恢复默认。

### 7.3 字段选择器

字段按基础资料、关联信息、金额、日期、状态、备注和系统追踪分组。支持：

- 搜索字段。
- 分组全选或清空。
- 恢复数据集默认字段。
- 拖动调整 Excel 列顺序。
- 显示字段是否可重新导入及敏感权限状态。

### 7.4 文件结构

支持两种结构：

1. 一个 Excel，多模块分别为工作表。
2. 一个 ZIP，包含数据导航 Excel、多个模块 Excel、附件目录和清单。

两种结构均支持自由选择部分大模块或全部模块。

### 7.5 快捷链接

- 单工作簿使用工作表内超链接和记录定位锚点。
- ZIP 使用相对路径链接数据导航、模块工作簿和附件。
- 选择人员和工资模块时，员工可以跳到工资明细，工资明细可以跳回员工。
- 项目、合同、发票、收付款、公司、班组和设备使用同一双向链接规则。

### 7.6 附件

默认只导出附件名称、分类、上传时间、大小和关联记录。

勾选“包含附件”后：

- 输出 ZIP。
- 原始附件按稳定目录结构保存。
- Excel 使用相对路径链接附件。
- 清单记录附件路径、大小和 SHA-256。

## 8. 导入设计

### 8.1 导入来源

1. 下载空白模板，填写后批量新增。
2. 系统导出后编辑，重新导入更新原记录。
3. 任意 Excel，通过字段映射导入。
4. 包含 Excel 和附件目录的 ZIP。

### 8.2 导入模式

- 仅新增。
- 仅更新。
- 新增和更新混合。

系统导出文件使用隐藏系统 ID 和并发版本定位原记录。空白模板使用业务编号创建新记录。任意 Excel 在映射阶段明确选择新增键和更新键。

### 8.3 字段映射

系统自动匹配完全相同、别名相同和历史映射列名。用户可以调整来源列到系统字段的映射，并保存为：

- 个人映射模板。
- 管理员共享映射模板。

映射只对当前数据集和版本有效。目标字段升级后，旧映射必须重新校验。

### 8.4 导入向导

1. 选择来源和导入模式。
2. 选择数据模块。
3. 上传 Excel 或 ZIP。
4. 识别工作表、列名和附件目录。
5. 确认字段映射。
6. 完整预览校验。
7. 显示新增、更新、未变化、关联影响和错误数量。
8. 全部通过后确认导入。

### 8.5 校验顺序

- 文件格式、文件大小、工作表和列标题。
- 必填、数据类型、长度、日期、数字和枚举。
- 文件内业务编号和系统 ID 重复。
- 系统 ID、业务编号和关联记录是否存在。
- 依赖数据集是否按顺序准备完成。
- 当前用户的模块、数据范围和敏感字段权限。
- 并发版本是否仍为最新。
- 模块业务规则和跨模块关系。
- 附件路径、扩展名、大小和 SHA-256。

### 8.6 整批原子性

任何错误、关联缺失、权限不足、并发冲突或附件不匹配都阻止整批写入。

导入执行流程为：

1. 原始文件保存到隔离暂存目录。
2. 全量解析和校验，不修改业务数据。
3. 附件完成安全检查和哈希验证。
4. 用户确认后，在数据库事务中按依赖顺序写入。
5. 数据库提交成功后，把附件从暂存区移动到正式目录。
6. 附件移动失败时执行补偿并标记任务失败，不显示虚假成功。

不允许通过 Excel 物理删除记录。停用、作废和结束日期使用现有业务字段和服务规则处理。

### 8.7 错误报告

错误报告可下载为 Excel，包含：

- 原始文件、工作表和行号。
- 来源列和目标字段。
- 原始值。
- 错误原因。
- 建议修正方式。
- 关联记录或冲突版本信息。

## 9. 页面设计

`/DataExchange` 分为三个视图：

### 9.1 导出数据

按模块、范围、字段、文件结构和附件选项组成一个连续工作区。字段目录不再单独占据页面首屏。

### 9.2 导入数据

使用分步向导呈现来源、上传、映射、校验、确认和结果。每一步只显示当前需要操作的控件。

### 9.3 任务记录

显示：

- 操作时间和操作人。
- 导入或导出类型。
- 模块、筛选条件和字段。
- 原始文件、结果文件和错误报告。
- 数据行数、附件数量和任务状态。
- 失败原因、完成时间和审计入口。

## 10. 权限与敏感数据

- 所有导出使用当前用户的数据范围和字段权限。
- 无敏感权限时，身份证、银行卡、工资和账户信息自动脱敏。
- 具备敏感信息权限的管理员可以选择完整字段。
- 全模块导出、含附件导出和全量导入更新仅限系统管理员或应用管理员。
- 共享字段模板和共享映射模板只有管理员可以发布。
- 导入导出记录操作人、时间、来源页面、模块、筛选、字段、行数和文件摘要。
- Excel 说明页标记敏感数据保管要求。

## 11. 后台任务与文件管理

大文件导入导出使用后台任务，不占用浏览器请求直到完成。

任务状态至少包含：等待、解析、校验、等待确认、执行、完成、失败和取消。任务文件保存在受控目录，不直接暴露服务器物理路径。下载使用授权端点和安全文件名。

文件名包含模块、范围、生成时间和任务编号。任务保留时间和文件清理策略由管理员配置。

## 12. 与统一财务账本的依赖

统一财务账本已有独立规格。财务 Excel 数据集必须在其核心实体、状态、分摊和对账模型稳定后实施。

本设计可以先交付数据集目录、页面、任务框架和稳定模块的数据交换。财务数据集随后接入最终模型，不为旧 `ReceivableEntry`、`PaymentEntry` 等结构额外建立长期双向模板。

## 13. 实施分期

1. 数据集目录、页面重构、筛选复用和后台任务基础。
2. 组织、公司、合作单位、员工、证书、项目、合同和工程量。
3. 工资、班组、设备、阶段成果和附件 ZIP。
4. 统一财务模型稳定后的财务、年度账和对账数据集。
5. 多模块组合、双格式、跨表链接和全量往返验证。

## 14. 测试与验收

至少覆盖：

- 每个用户可维护字段已登记为可交换或有明确排除原因。
- 当前视图导出复用页面筛选、排序、字段和字段顺序。
- 最后一次选择按用户、页面和数据集隔离保存并恢复。
- 全量、自定义字段、多模块和全部模块导出。
- 单工作簿和 ZIP 两种格式。
- 跨表、跨工作簿和附件相对链接。
- 空白模板新增、系统导出更新和任意 Excel 映射。
- 个人/共享字段模板和映射模板。
- 类型、必填、关联、权限、并发和附件校验。
- 任意错误时整批不写入。
- 敏感字段脱敏和完整字段权限。
- 从全字段导出文件导入空测试库后，业务字段和关联关系一致。
- 代表性大数据量导入导出性能。
- 桌面与移动端页面无重叠和页面级横向溢出。

## 15. 成功标准

- 用户可以把系统全部业务数据按 Excel 形式保存到电脑、编辑和分享。
- 用户可以自由选择模块、数据行、字段和字段顺序。
- Excel 可以安全地新增或更新系统数据，而不产生部分成功或静默覆盖。
- Excel 和附件 ZIP 中的关联能够快速互相访问。
- 任何模块的导入导出都不产生新的业务数据副本。
