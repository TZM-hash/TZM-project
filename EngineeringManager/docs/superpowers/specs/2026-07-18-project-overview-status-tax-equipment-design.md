# 项目总览状态、税金与重要施工机械设计

## 目标

调整项目主档与项目总览，使项目阶段同时承担归档语义，独立维护合同签订进度和项目允许使用的税金组合，并从施工详情的同一数据源聚合重要施工机械。发票明细必须使用项目已配置的开票公司、税率和发票类型，项目备注移动到总览末尾并单独占一整行。

## 范围

本次包含：

- 项目阶段精简与旧数据迁移。
- 删除独立归档状态。
- 新增项目合同签订状态。
- 新增独立的项目税金配置明细。
- 发票开票公司、税率和发票类型的选择与校验。
- 施工详情设备记录的“显示在项目总览”开关。
- 项目总览重要施工机械聚合与定位跳转。
- 项目备注的布局调整。
- 项目列表、编辑页面、显示文本、导出和相关测试的同步调整。

本次不包含：

- 根据合同记录自动推导合同签订状态。
- 根据发票历史自动改变项目税金配置。
- 为施工班组增加项目总览展示开关。
- 新增税率范围之外的自定义税率。
- 因项目归档而锁定项目或财务数据。

## 领域模型

### 项目阶段

`ProjectStage` 只保留以下五个业务状态：

1. 待进场
2. 施工中
3. 停工中
4. 已完工未结算
5. 已结算归档

`已结算归档` 同时表达归档完成，不再保留独立的 `ArchiveStatus` 业务字段或页面控件。该阶段不锁定数据，具备管理权限的用户仍可修改项目及其明细。

### 合同签订状态

项目新增独立的 `ContractSigningStatus`：

1. 未签合同
2. 合同已寄出
3. 合同已签完

合同签订状态由管理员手动维护。新增、修改或删除实际合同记录均不自动改变该状态。

### 项目税金配置

新增项目税金配置明细实体。每条配置属于一个项目，并包含：

- 税率：仅允许 1%、3%、6%、9%、13%。
- 发票类型：普票或专票。
- 是否启用。
- 创建时间、更新时间及并发标记等项目现有审计所需信息。

同一项目内，“税率＋发票类型”组合必须唯一。一个项目可以同时启用多个组合，也允许同一税率同时配置普票和专票，例如 `3% 普票` 与 `3% 专票`。

税金配置是发票录入的允许范围，不根据已有发票反向推导。项目可以暂时不配置税金，但录入发票前必须至少存在一个启用组合。

已被发票使用的税金配置不得物理删除，只能停用。停用不影响历史发票查看，但不能用于新发票；修改历史发票时必须重新选择当前项目的启用组合。

### 发票

每张发票必须选择：

- 当前项目的一家签约公司作为开票公司。
- 当前项目的一条启用税金配置，从而确定税率和发票类型。

发票服务层同时校验项目、开票公司和税金配置的归属关系，不能只依赖页面下拉框。发票表格继续分别显示税率和发票类型，以便筛选、导出和倒查。

旧发票中可识别的“普票/专票”继续保留。历史票种为空或无法识别时显示“未设置”，不擅自推断；再次编辑该发票时必须补选当前项目的有效税金组合。

### 施工机械总览标记

`ProjectConstructionRecord` 新增 `ShowInProjectOverview` 布尔字段。该字段属于具体项目下的具体施工记录，而不是设备主档，因为同一设备在不同项目或不同进退场批次中的重要性可能不同。

只有 `Equipment` 类型的施工记录允许开启此字段。`ConstructionCrew` 类型在页面上不显示该控件，服务层也拒绝班组记录提交为开启状态。

项目总览直接关联查询施工详情记录，不复制设备名称、编号或进退场信息，从而保证两个页面的数据源一致。

## 旧数据迁移

### 项目阶段迁移

旧阶段映射如下：

| 旧状态 | 新状态 |
| --- | --- |
| 前期跟踪、待签合同、待进场 | 待进场 |
| 施工中 | 施工中 |
| 已停工 | 停工中 |
| 完工待验收、结算中、质保期 | 已完工未结算 |
| 已关闭 | 已结算归档 |

若旧项目的独立归档状态为“已归档”，无论旧阶段为何，都迁移为“已结算归档”。旧归档状态为“待归档”但尚未归档的项目迁移为“已完工未结算”。完成映射后删除独立归档状态列。

### 合同签订状态初始化

- 已存在至少一条合同记录的项目初始化为“合同已签完”。
- 尚无合同记录的项目初始化为“未签合同”。
- 不自动初始化为“合同已寄出”，该状态由管理员后续手动维护。

### 税金与发票迁移

项目税金配置不根据历史发票自动生成，以免把历史录入错误固化为项目配置。现有项目初始允许没有税金配置，管理员在新增或修改发票前补充允许组合。

可识别的历史发票类型转换为结构化发票类型；空值或无法识别的历史值保留为“未设置”的兼容状态。新建发票和再次编辑旧发票时不允许保存未设置状态。

### 施工机械迁移

现有施工详情记录的 `ShowInProjectOverview` 默认设为 `false`，避免迁移后把所有机械突然显示到项目总览。管理员按项目逐条勾选重要机械。

## 应用服务与数据流

### 项目读取

项目工作台服务返回：

- 新项目阶段。
- 合同签订状态。
- 项目全部税金配置及启用状态。
- 已勾选的重要设备施工记录。

重要机械记录包含施工记录标识、设备标识、设备编号、设备名称、进场日期、退场日期和当前在场状态。总览点击机械时携带施工记录标识，切换到施工详情页签并定位对应记录。

### 项目保存

项目快捷编辑和详细编辑统一提交项目主档字段及税金配置集合。服务层负责：

- 验证项目阶段和合同签订状态合法。
- 验证税率在固定集合中。
- 验证票种为普票或专票。
- 拒绝重复组合。
- 对已使用配置执行停用而非删除规则。
- 保留现有并发控制、修改原因和审计日志。

### 发票保存

发票新增和修改统一执行：

1. 项目存在且可用。
2. 开票公司属于该项目的签约公司。
3. 所选税金配置属于该项目。
4. 所选税金配置当前启用。
5. 发票保存的税率和票种与所选配置一致。

详细财务录入页面和项目详情原位编辑页面必须调用同一服务校验，避免入口之间产生不一致。

### 施工详情保存

设备施工记录保存时同步保存 `ShowInProjectOverview`。项目间自动生成的后续草稿施工记录默认不继承该标记，避免新项目未经确认就展示设备。班组记录的该值必须为 `false`。

## 页面设计

### 项目总览

沿用现有四列紧凑卡片布局，不引入新的视觉体系：

- 删除“归档状态”。
- “项目阶段”只显示五个新状态。
- 新增“合同签订”卡片。
- 新增“税金”卡片，查看状态使用标签展示，例如 `3% 专票`、`9% 普票`。
- 税金编辑使用五行税率矩阵，每个税率分别提供“普票”和“专票”勾选项。
- 新增“施工机械”区域，只展示已勾选的重要设备施工记录。
- 每条机械展示设备编号、设备名称、进场日期，以及退场日期或“仍在场”。
- 同一设备多个被勾选批次按施工记录分别展示。
- 机械条目可跳转并定位到施工详情中的对应记录。
- 项目备注移动到总览最后，单独占满一整行，查看时保留换行，编辑时使用较高的多行文本框。
- 现有金额、收付款比例和开票比例保持原口径。

### 项目详细编辑

详细编辑页面同步删除归档状态，提供五项项目阶段、三项合同签订状态和同样的税金组合矩阵。项目备注继续位于基础资料区域的最后并占整行。

### 发票明细

新增和编辑发票时：

- 开票公司下拉框仅列出当前项目签约公司。
- 税金组合下拉框仅列出当前项目已启用配置，文本格式为“税率 · 票种”。
- 发票列表分别显示开票公司、税率和发票类型。
- 若项目没有启用税金配置，页面提示先维护项目税金配置，服务层同样拒绝保存。

### 施工详情

设备类型记录显示“显示在项目总览”复选框；切换为班组类型时隐藏并清空该值。列表增加是否在总览显示的状态提示，并为从总览跳转过来的施工记录提供定位高亮。

### 响应式布局

桌面端保持四列信息密度。窄屏下卡片与税金矩阵按现有响应式规则堆叠，项目备注始终独占整行，机械条目不得产生横向溢出。验收宽度包括 390px。

## 错误处理

- 非法或重复税金组合返回明确的业务错误。
- 发票开票公司不属于项目时拒绝保存。
- 发票税金配置不属于项目、已停用或不存在时拒绝保存。
- 删除已被发票使用的税金配置时提示改为停用。
- 班组记录开启项目总览标记时拒绝保存。
- 并发冲突继续使用现有 `ConcurrencyStamp` 机制提示刷新重试。
- 所有管理操作继续记录修改原因和审计前后值。

## 测试与验收

### 自动化测试

- 迁移测试覆盖旧阶段、旧归档状态和合同状态初始化规则。
- 项目服务测试覆盖五种阶段、合同签订状态、多税金组合、重复组合和停用规则。
- 发票服务测试覆盖开票公司归属、税金配置归属、启用状态和历史未设置票种兼容。
- 施工服务测试覆盖设备标记保存、班组拒绝、自动流转草稿不继承标记。
- 工作台服务测试覆盖只聚合已勾选机械及完整展示字段。
- Razor 页面测试覆盖删除归档状态、新增合同与税金控件、发票字段、机械开关、备注末行布局。
- 项目列表、显示文本、导出和权限测试同步更新。

### 手工验收

- 项目快捷编辑与详细编辑均可维护阶段、合同状态和多个税金组合。
- 没有税金配置时不能录入发票，并得到明确提示。
- 每张发票可独立选择项目签约公司及允许的税金组合。
- 停用组合不影响历史发票，但新发票无法选择。
- 施工详情勾选设备后总览即时出现，取消后消失。
- 总览机械点击后正确定位施工记录。
- 项目备注位于总览最后、占满整行并保留换行。
- 桌面端和 390px 页面均无内容遮挡或横向溢出。

### 质量门禁

数据库迁移前运行 `EngineeringManager/scripts/verify-backup-restore.ps1`。实现完成后运行定向测试、完整测试套件和仓库现有质量门禁，并确认工资、员工、班组、财务和项目既有流程无回归。
