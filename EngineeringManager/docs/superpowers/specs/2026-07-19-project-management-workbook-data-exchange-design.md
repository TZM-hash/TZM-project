# 项目管理业务工作簿 Excel 数据交换设计

**日期：** 2026-07-19  
**状态：** 已完成业务讨论，待用户审阅书面规格  
**范围：** 首期只实现项目管理，其他大类暂不迁移

## 1. 目标

在现有 Excel 数据交换基础上，建立“一个业务大类对应一个工作簿”的项目管理数据交换能力。用户可以：

- 按项目筛选和手动勾选项目。
- 导出全部项目、当前筛选结果或指定项目。
- 导出全部项目细项，或只导出勾选的细项。
- 把多个项目集中放在同一工作簿中，每个细项使用独立工作表。
- 导出单个项目时使用与跨项目导出相同的工作簿结构。
- 导入系统导出的工作簿、空白模板或任意来源 Excel。
- 在同一工作簿中新增或更新项目及其合同、工程量、财务和其他关联明细。
- 在确认前完成全量校验，任何错误都不会产生部分业务数据。
- 可选地将项目附件和阶段成果附件作为 ZIP 文件本体交换。

## 2. 范围与边界

### 2.1 首期覆盖

项目管理工作簿覆盖：

1. 项目主档
2. 项目经营汇总
3. 合同
4. 工程量
5. 里程碑
6. 项目人员
7. 项目合作单位
8. 施工详情
9. 阶段成果
10. 应收
11. 收款
12. 应付
13. 付款
14. 发票
15. 附件清单（选择包含附件时）

项目经营汇总为计算结果，只导出、不导入。附件清单交换附件元数据和关联信息；附件文件本体只在 ZIP 模式下交换。

### 2.2 不在首期范围

- 员工、工资、设备、合作单位等其他大类的工作簿迁移。
- 通过缺少 Excel 行或工作表物理删除业务记录或附件。
- 用 Excel 直接写数据库或绕过现有业务服务、权限和并发校验。
- 将 Excel 作为完整系统备份格式。

首期允许抽取可复用的工作簿定义、任务、预览、文件和事务基础，但不迁移其他大类的页面和业务适配器。

## 3. 核心概念

### 3.1 两个独立选择维度

导出范围由两个维度组成：

- **项目范围：** 全部授权项目、当前筛选结果、手动勾选项目。
- **细项范围：** 全部子表，或用户勾选的一个/多个子表。

两者可以任意组合，例如：全部项目 + 全部子表、单项目 + 全部子表、筛选结果 + 仅收款、手动勾选项目 + 合同和工程量。

“全选当前筛选结果”按跨分页的全部命中项目计算，不只选择当前页面。

### 3.2 安全合并导入

全量导入指处理上传文件中全部已选工作表和数据行，不表示用 Excel 覆盖系统现状：

- 文件中的新记录新增。
- 带系统 ID 或业务键的已有记录更新。
- Excel 没有出现的系统记录保持不变。
- 停用、作废和结束状态通过业务字段处理。
- 不因缺少工作表、数据行或附件行而删除系统数据。

## 4. 总体架构

### 4.1 业务工作簿层

在通用数据交换层之上增加业务工作簿抽象。每个工作簿定义声明：

- 工作簿名称和版本。
- 可选子表、稳定数据集键和中文表名。
- 字段目录、数据类型、可导入/可导出属性和技术列。
- 子表依赖顺序和跨表关联键。
- 项目筛选器和手动勾选规则。
- 业务权限、敏感字段和并发要求。
- 标准模板、标准导入和任意来源映射导入方式。
- 导出器、解析器、校验器和写入适配器。

首期实现项目工作簿适配器。以后其他大类只需提供自己的工作簿定义和业务适配器。

### 4.2 通用数据交换层

通用层负责：

- XLSX 多工作表和 ZIP 读写。
- 目录、说明、版本和任务元数据。
- 导入预览、错误报告和任务历史。
- 单工作簿整批原子事务协调。
- 文件临时存储、授权下载和清理。
- 附件路径安全、大小、扩展名和哈希校验基础能力。

页面不直接生成业务数据，也不在页面中复制项目导出逻辑。

## 5. 工作簿结构

### 5.1 工作表

项目工作簿使用稳定的工作表键和中文名称：

1. `目录/说明`
2. `项目主档`
3. `项目经营汇总`
4. `合同`
5. `工程量`
6. `里程碑`
7. `项目人员`
8. `项目合作单位`
9. `施工详情`
10. `阶段成果`
11. `应收`
12. `收款`
13. `应付`
14. `付款`
15. `发票`
16. `附件清单`（仅在包含附件时）

`目录/说明`始终存在。选择全部细项时生成所有业务表和只读汇总表；选择部分细项时只生成用户勾选的明细表，汇总表只有被明确勾选时才生成。

### 5.2 业务列和技术列

每张明细表是一张普通矩形数据表，不依赖合并单元格和 Excel 公式。业务列包括项目编号、合同编号、日期、金额、状态、备注等可读字段。

技术列默认隐藏并保护，包括：

- 当前记录系统 ID。
- 所属项目系统 ID。
- 必要时的父记录系统 ID，例如合同 ID、应收记录 ID或阶段成果 ID。
- 并发版本。
- 数据集版本和工作簿版本标识。

导入服务不能信任隐藏状态，必须重新检查技术列值、权限、关联和并发版本。空白模板可以隐藏系统 ID 但保留业务关联键，便于新增记录。

### 5.3 汇总表

`项目经营汇总`包含合同额、工程量汇总、应收、已收、未收、应付、已付、未付、已开票、未开票等计算结果，以及统计截止日和筛选说明。

汇总字段不参与导入。用户修改合同、工程量、收款、付款或发票明细后，系统重新计算项目经营汇总。

## 6. 项目选择与页面入口

### 6.1 项目列表页

项目列表复用现有筛选和保存视图，包括关键词、项目状态、签约公司、负责人、合作方式、金额区间等条件。用户可以：

- 手动勾选项目。
- 全选当前筛选结果。
- 选择全部或部分项目细项。
- 在导出前看到筛选命中数和最终选择数。

项目列表原有的轻量当前视图导出保留；项目工作簿导出使用新的多表工作簿流程。

### 6.2 项目详情页

项目详情页以当前项目为范围，可以选择全部细项或当前关注的一个/多个细项。单项目文件与跨项目文件使用相同表结构。

### 6.3 导入导出中心

导入导出中心可以重新筛选、搜索和手动勾选项目，并选择细项范围。由项目列表或详情页进入时，自动带入来源页面的项目范围和筛选条件。

所有入口调用同一个项目工作簿服务。

## 7. 导出流程

1. 用户选择项目范围和细项范围。
2. 服务根据当前用户权限重新解析最终项目 ID 集合，不信任页面提交的未授权 ID。
3. 工作簿定义按细项顺序生成目录、说明、业务表和可选汇总表。
4. 业务明细按项目编号和稳定排序写入；技术 ID和并发版本写入隐藏列。
5. 勾选包含附件时，生成 ZIP：
   - `project-workbook.xlsx`
   - `attachments/...` 附件文件
   - `manifest.json`
   - `checksums.sha256`
6. `附件清单`记录附件系统 ID、关联项目/合同/阶段成果、原文件名、分类、类型、大小、上传时间、相对路径和 SHA-256。
7. 导出任务记录操作人、筛选条件、项目数量、子表、附件选项、文件摘要和结果。

全量项目包括用户有权看到的进行中、已完工和已停用项目。筛选和手动勾选可以限制最终范围。

## 8. 导入流程

### 8.1 来源

- 系统导出的标准项目工作簿。
- 系统生成的空白项目模板。
- 任意来源 Excel，通过映射导入。
- 含附件的项目 ZIP。

标准工作簿按固定表名和元数据自动识别；任意 Excel 先选择目标子表，再配置字段映射和项目/合同等关联字段。

### 8.2 向导

1. 选择来源和导入模式：仅新增、仅更新、新增或更新。
2. 上传文件并识别工作表、版本、附件目录和清单。
3. 选择要处理的子表；未上传或未选择的子表不参与此次导入。
4. 对任意来源 Excel 确认字段映射、业务键和关联解析方式。
5. 预览所有子表的总行数、新增、更新、未变化、跳过和错误数量。
6. 展示关联解析、权限、数据类型、枚举、必填、金额、日期、并发和附件校验结果。
7. 所有子表通过后确认整批导入。
8. 展示结果并提供错误报告或导入结果报告。

### 8.3 关联和写入顺序

完整工作簿按以下依赖顺序处理：

`项目主档 → 项目关联 → 合同 → 工程量 → 阶段成果 → 项目财务 → 其他项目明细 → 附件`

同一工作簿可以同时新增项目、合同、工程量和财务明细。跨表新增优先使用文件中的临时业务键或系统 ID建立映射；单独导入子表时，父级记录必须已经存在。

### 8.4 空白、更新和删除规则

- 文件中没有某一列：不修改该字段。
- 文件中有该列且单元格为空：清空可为空字段。
- 必填字段为空：报错。
- 计算字段、只读字段和未映射字段：不更新。
- 任意来源映射流程提供“空白时不更新”选项，默认关闭。
- 缺少工作表、数据行或附件行不触发删除。
- 不允许通过 Excel 物理删除业务记录或附件。

### 8.5 原子性

整份上传文件是一个原子批次：

- 任意表或任意行有错误时不能确认。
- 所有表在同一个数据库事务中按依赖顺序写入。
- 附件先在隔离暂存区完成安全检查和哈希校验。
- 数据库提交成功后移动附件到正式存储。
- 附件移动失败时执行补偿并将任务标记为失败，不显示虚假成功。

## 9. 权限、并发和安全

- 导出和导入服务重新执行项目范围授权检查。
- 每张子表检查自身业务权限；选择任何无权子表都会阻止整批导入。
- 导入要求对所有选中子表同时拥有维护权限。
- 敏感字段根据现有敏感信息权限脱敏、隐藏或拒绝访问。
- 含附件导出、全量项目工作簿和跨表新增导入保留管理员/业务管理员限制。
- 技术列和并发版本用于防止旧文件静默覆盖新数据；版本冲突必须在预览中显示。
- 文件上传限制大小、扩展名、ZIP 路径穿越、恶意文件名和附件哈希。
- 工作簿版本、数据集版本或字段规则不兼容时，进入兼容检查并阻止直接写入。
- 任务记录操作人、时间、筛选条件、项目范围、子表、文件摘要、附件数量和结果。

## 10. 组件边界

建议组件边界如下：

- **业务工作簿定义：** 声明项目子表、字段、版本、依赖和权限。
- **项目范围解析器：** 复用项目列表筛选和授权逻辑，生成最终项目 ID集合。
- **项目工作簿导出适配器：** 查询项目主档、子表、财务和附件，生成表格行。
- **项目工作簿导入适配器：** 解析标准表和映射表，构造跨表暂存模型。
- **通用工作簿协调器：** 处理文件识别、预览、任务、事务和结果。
- **附件交换器：** 处理附件清单、暂存、哈希和正式存储移动。
- **项目数据写入服务：** 调用现有项目、财务、阶段成果和施工业务服务，保持业务规则一致。

页面只负责选择、上传、显示预览和确认，不直接操作实体或拼装业务数据。

## 11. 错误报告和任务状态

错误报告至少包含：

- 文件、工作表、行号和列名。
- 目标数据集和字段。
- 原始值。
- 错误原因和修正建议。
- 关联记录、并发版本或附件路径信息。

任务状态至少覆盖：等待、解析、校验、等待确认、执行、完成、失败和取消。任务历史显示操作人、时间、项目范围、筛选、子表、行数、附件数量、结果文件和错误报告。

## 12. 测试与验收

### 12.1 单元测试

- 工作簿子表目录、字段顺序、版本和依赖顺序。
- 项目筛选、跨分页全选和手动勾选合并。
- 技术 ID、业务键和并发版本解析。
- 空白单元格、必填项、枚举、金额、日期和状态校验。
- 跨表临时键和父级依赖解析。
- 附件路径、大小、扩展名和 SHA-256 校验。
- 汇总字段只读和细项范围解析。

### 12.2 集成测试

- 全部项目、筛选结果、手动项目和单项目导出。
- 全部细项、单细项和多细项工作簿。
- 标准工作簿、空白模板和任意 Excel 映射导入。
- 新项目到合同、工程量、财务明细的跨表新增。
- 混合新增和更新、并发冲突、权限不足和关联缺失。
- 任意表错误时整份事务不写入。
- ZIP 附件导出、导入、哈希和文件关联。
- 导出后修改再导入的往返一致性。

### 12.3 回归和页面验收

- 现有项目列表导出、项目筛选、分页和保存视图不回归。
- 其他大类现有导入导出行为不改变。
- 项目管理页面在桌面和移动端无重叠、无页面级横向溢出。
- 未授权项目、敏感字段和附件不会通过工作簿泄露。

## 13. 实施顺序

1. 固定项目工作簿定义、数据集键、字段目录和版本。
2. 抽取通用多工作表工作簿协调和整批预览能力。
3. 实现项目范围解析和项目列表/详情/导入导出中心入口。
4. 实现项目主档、合同、工程量、里程碑、人员、合作单位、施工、阶段成果和项目财务子表导出。
5. 实现标准工作簿和空白模板导入、跨表依赖和整批事务。
6. 实现任意来源 Excel 映射导入。
7. 实现附件清单、ZIP、哈希校验和补偿流程。
8. 完成回归、往返、权限、并发、附件和代表性数据量测试。

## 14. 成功标准

用户可以在项目列表或详情页筛选、勾选项目，导出全部或部分项目细项到统一结构的 Excel 工作簿；也可以编辑标准工作簿后安全回导，或通过映射导入旧 Excel。完整工作簿支持跨表新增和更新，任意错误都不会产生部分成功，项目经营汇总由明细自动重算，附件能够随 ZIP 一起安全归档和恢复。其他大类无需在本阶段改动即可继续使用现有功能。
