# 系统备份、定时策略与跨服务器恢复设计

**日期：** 2026-07-19  
**状态：** 已完成业务讨论，待用户审阅书面规格

## 1. 目标

在现有手动 SQL Server 数据库备份和附件 ZIP 基础上，建立可配置、可审计、可验证、可跨服务器恢复的系统备份体系。

系统需要支持：

- 系统设置备份和完整数据备份两种类型。
- 两种类型分别手动执行或定时执行。
- 按间隔或固定时间配置计划。
- 分别配置启用状态、保留数量、本机目录和 NAS 目录。
- 本机备份和局域网共享目录/NAS 副本。
- 备份下载、校验、删除、失败提醒和任务历史。
- 页面内设置恢复。
- 独立完整恢复工具。
- 在另一台服务器安装兼容系统后，通过备份包恢复数据库、附件、设置和关系。

## 2. 当前现状

当前系统已经支持：

- 管理员手动创建备份。
- SQL Server `.bak` 文件。
- 附件目录 `.zip` 文件。
- 备份任务状态、路径、错误和时间记录。
- 备份恢复验证脚本和部署文档。

当前缺少：

- 设置备份类型。
- 定时计划和保留策略。
- NAS 副本和目标状态。
- 备份包清单、版本和哈希校验。
- 页面下载、验证和删除操作。
- 设置恢复向导。
- 完整数据恢复工具。
- 跨服务器恢复流程和自动验收。

## 3. 备份类型

### 3.1 设置备份

设置备份是轻量、可预览、可选择恢复的配置包，包含：

- 组织架构、部门和分支机构。
- 业务年度和基础分类。
- 系统显示设置。
- 用户账号基本资料、启用状态、角色和权限配置。
- 备份计划和保留策略。
- 其他用户可维护的系统级配置。

设置备份不包含：

- 明文密码。
- 登录令牌。
- 连接字符串。
- TLS 证书和私钥。
- 服务器密钥。
- 业务数据和附件。

设置恢复后，无法安全保留认证状态的账号标记为需要重置密码。完整数据库恢复仍可以保留数据库中的原认证信息。

### 3.2 完整数据备份

完整备份用于灾难恢复和跨服务器迁移，包含：

- SQL Server 完整数据库备份。
- 全部附件原始文件。
- 系统设置快照。
- 应用版本和数据库迁移版本。
- 文件清单、大小和 SHA-256。
- 备份时间、来源系统标识和任务信息。
- 恢复说明和环境要求。

数据库备份保留业务数据、账号权限、审计记录、提醒记录和内部关系。

## 4. 备份包格式

### 4.1 设置包

设置包使用 ZIP，包含：

- `manifest.json`。
- 分类别设置 JSON。
- 用户/角色/权限清单。
- `checksums.sha256`。
- 恢复说明。

### 4.2 完整包

完整包使用稳定目录结构：

```text
EngineeringManager_Backup_<timestamp>_<taskId>/
  manifest.json
  database/EngineeringManager.bak
  attachments/...
  settings/settings-snapshot.zip
  checksums.sha256
  restore-readme.txt
```

最终可以压缩为单一可下载文件。包内使用相对路径，不保存旧服务器的绝对目录。

### 4.3 不加密

按已确认要求，备份包不加密。安全依赖：

- 本机目录 ACL。
- NAS 共享权限。
- 管理员下载和恢复授权。
- SHA-256 完整性校验。
- 下载、删除和恢复审计。
- 页面明确提示备份包含敏感数据。

## 5. 备份策略

设置备份和完整数据备份分别拥有独立策略：

- 启用或停用。
- 手动执行。
- 按间隔执行。
- 固定时间执行。
- 时区。
- 本机目标目录。
- NAS 目标目录。
- 本机保留数量。
- NAS 保留数量。
- 失败提醒。
- 最近执行时间和下次执行时间。
- 遗漏任务补执行规则。

策略持久化保存，应用重启后不丢失。

## 6. 调度与执行

备份不在页面请求中同步完成。当前 Windows Server + IIS 环境采用持久化计划记录和独立执行器：

- 页面只保存策略和创建手动任务。
- Windows 计划任务或后台执行器定期检查到期任务。
- 执行器领取任务并使用任务锁防止重复执行。
- 应用重启或服务器短暂停机后，执行器检查遗漏任务并按策略补执行。
- 同一备份类型同一时间只允许一个运行任务。

后续迁移到云服务器时，可以替换调度宿主，但继续使用同一策略和任务服务。

## 7. 存储目标

当前阶段采用：

1. 当前服务器本机目录。
2. 可配置的局域网共享目录或 NAS。

执行顺序：

1. 在本机临时目录生成。
2. 校验完整性。
3. 原子移动到本机正式备份目录。
4. 复制到 NAS 临时文件名。
5. NAS 校验通过后改为正式文件名。

本机成功但 NAS 复制失败时，任务状态为“本机成功、异地副本失败”，保留本机文件并创建失败提醒。不得把它显示为完整成功。

云对象存储暂不实现，正式版发布后再增加目标适配器。

## 8. 保留与清理

- 设置备份和完整备份分别配置保留数量。
- 本机和 NAS 分别计算保留策略。
- 只有新备份完成并校验后才清理旧备份。
- 正在下载、恢复或校验的文件不能删除。
- 手动标记“保留”的备份不参与自动清理。
- 自动清理和手动删除都写审计日志。
- 清理失败创建提醒，但不影响新备份完成状态。

## 9. 页面设计

`/Backups` 分为：

### 9.1 备份总览

显示：

- 最近一次设置备份。
- 最近一次完整备份。
- 下次计划时间。
- 本机目录和 NAS 连接状态。
- 可用空间。
- 最近失败和部分成功任务。

### 9.2 设置备份

提供立即备份、下载、校验、删除和设置恢复入口。

### 9.3 完整数据备份

提供立即备份、下载、校验、删除和生成恢复任务入口。

### 9.4 定时策略

设置备份和完整备份分别配置启用、执行方式、间隔/时间、目标目录、保留数量和失败提醒。

### 9.5 恢复记录

显示上传、校验、预览、执行、完成和失败记录，以及恢复前安全备份和验收结果。

## 10. 设置恢复

设置备份可以在页面中恢复：

1. 上传或选择设置备份包。
2. 校验清单、版本和 SHA-256。
3. 读取当前设置并生成差异。
4. 按组织、业务年度、分类、显示设置、用户角色权限和备份策略选择恢复类别。
5. 显示新增、更新、跳过和冲突项目。
6. 管理员确认后，在事务中恢复。
7. 记录恢复前后快照和操作原因。

设置恢复不修改业务数据、附件、连接字符串、TLS 或服务器密钥。

## 11. 完整数据恢复

### 11.1 为什么使用独立恢复工具

应用运行时不能可靠覆盖自身正在使用的 SQL Server 数据库，因此完整恢复不在普通 Razor 页面请求内执行。

页面负责：

- 上传或登记备份包。
- 完成初步校验。
- 生成恢复任务和恢复指令。

独立恢复工具负责实际恢复。

### 11.2 恢复前检查

- 备份包结构和 SHA-256。
- 应用版本和数据库迁移兼容性。
- SQL Server 版本和数据库名称。
- 磁盘空间和附件目录权限。
- 新服务器连接字符串已配置。
- TLS 和服务器密钥已配置。
- 当前没有运行中的导入、导出、备份或恢复任务。
- 管理员明确确认维护窗口。

### 11.3 恢复流程

1. 进入维护模式，拒绝新的业务写入。
2. 创建当前环境的恢复前安全备份。
3. 停止应用或释放数据库连接。
4. 恢复 SQL Server 数据库。
5. 恢复附件到临时目录。
6. 校验附件清单后原子切换正式目录。
7. 应用必要的兼容迁移。
8. 启动应用。
9. 验证健康端点、迁移版本和管理员登录。
10. 验证关键表数量、金额汇总、附件数量和关联完整性。
11. 退出维护模式。

任一步失败时保留恢复日志，并优先使用恢复前安全备份回滚。

### 11.4 跨服务器迁移

新服务器需要先安装兼容应用、SQL Server 和 IIS，配置新的连接字符串、TLS 和服务器密钥。之后选择完整备份包执行恢复。

旧服务器绝对路径不会覆盖新服务器路径。数据库、附件和设置关系保持不变。

## 12. 权限与审计

- 只有系统管理员和应用管理员可以查看备份页面。
- 完整恢复使用独立的高权限操作确认。
- 创建、下载、校验、删除、策略修改和恢复都写审计。
- 审计包含操作人、时间、备份类型、文件摘要、来源和目标、原因、结果和错误。
- 页面不显示数据库或附件的真实服务器路径给无权限用户。

## 13. 错误处理与提醒

提醒覆盖：

- 数据库备份失败。
- 附件打包失败。
- 本机空间不足。
- NAS 不可用或复制失败。
- SHA-256 不匹配。
- 定时任务未执行。
- 自动清理失败。
- 设置恢复冲突。
- 完整恢复失败或回滚失败。

任务状态至少区分：等待、运行、本机成功、异地副本失败、完成、失败和取消。

## 14. 实施分期

1. 扩展备份任务、包清单、下载/校验/删除和页面总览。
2. 设置备份生成、差异预览和分类恢复。
3. 两类备份策略、持久化调度、保留清理和失败提醒。
4. 本机 + NAS 双目标复制和部分成功状态。
5. 独立完整恢复工具、维护模式和恢复前安全备份。
6. 一次性测试环境中的完整恢复与跨服务器演练。

## 15. 测试与验收

至少覆盖：

- 设置包包含全部已确认配置且不包含明文密码或服务器密钥。
- 完整包包含数据库、附件、设置、版本、清单和 SHA-256。
- 手动、间隔和固定时间计划。
- 设置备份和完整备份的策略隔离。
- 应用重启后的遗漏任务补执行和任务锁。
- 本机写入、NAS 复制、原子文件名和部分成功状态。
- 保留数量、手动保留和清理审计。
- 备份下载、校验损坏和管理员授权。
- 设置差异预览、分类恢复、冲突和事务回滚。
- 完整恢复前检查、维护模式和恢复前安全备份。
- 数据库、附件、账号权限、审计和关联关系恢复。
- 恢复后的关键金额汇总和附件数量一致。
- `/health/live`、`/health/ready` 和管理员登录通过。
- 恢复演练只在一次性测试数据库和临时附件目录执行，不连接生产数据库。

## 16. 成功标准

- 系统能够按策略持续生成本机和 NAS 备份，并明确报告每个目标的状态。
- 管理员可以手动备份、下载、校验、删除和恢复设置。
- 完整备份包可以在另一台兼容服务器恢复全部系统数据和附件。
- 恢复过程可验证、可回滚、可审计，不依赖 Excel 重建关系。
