# 项目总览紧凑快捷编辑与合作单位汇总设计

## 目标

- 将项目总览快捷编辑中的签约公司和税金设置改为类似“列管理”的下拉复选菜单，避免展开编辑时撑高总览面板。
- 工程金额口径只允许在快捷编辑状态下切换，查看状态不显示可操作控件。
- 移除项目里程碑展示，只保留项目人员和项目合作单位。
- 项目合作单位统一展示相关总包、施工班组和项目合作方。

## 交互设计

### 签约公司

- 查看状态继续显示当前已选签约公司名称，未选择时显示“未设置”。
- 快捷编辑状态显示下拉复选菜单，摘要显示“签约公司”和“已选 N 项”。
- 菜单内列出全部可选自有公司，当前配置默认勾选。
- 支持“全选”和“清空”，最终仍通过现有 `QuickEdit.LegalEntityIds` 提交，不改变服务契约。

### 税金

- 查看状态继续显示已配置的税率与发票类型标签。
- 快捷编辑状态显示下拉复选菜单，摘要显示“税率与发票类型”和“已选 N 项”。
- 选项按税率分组，每项表示一个“税率 + 发票类型”组合。
- 支持“全选”和“清空”，最终仍通过现有 `QuickEdit.TaxConfigurationSelections` 提交。

### 工程金额

- 查看状态只显示项目阶段决定的口径名称及统一工程金额，不呈现下拉框。
- 快捷编辑状态才显示“暂估金额 / 结算金额”下拉框。
- 下拉框只切换显示口径，不产生第二套金额，也不允许手工修改自动计算金额。
- 取消快捷编辑后恢复项目阶段对应的默认口径。

## 关联资料设计

- 删除项目里程碑面板和空状态。
- 项目人员面板保持现有数据和展示方式。
- 项目合作单位面板汇总并去重以下来源：
  - 项目总览的总包单位，角色显示“总包”。
  - 当前项目施工记录中的施工班组，角色显示“施工班组”。
  - 有效的 `ProjectPartner` 关联，角色沿用甲方/总包、施工班组、材料供应商或零星供应商。
- 同一单位在多个来源中出现时合并为一项，并合并角色标签；优先保留项目合作方备注。
- 合作单位数量显示最终去重后的条目数。
- 不新增数据库字段，不改变项目、班组或合作方的既有关系。

## 实现边界

- 复用现有 `selection-dropdown` 和 `data-check-selector` 控件样式与交互。
- 为选择控件补充通用“清空”动作，不影响现有全选和恢复默认行为。
- 合作单位汇总在项目详情展示层完成，使用现有工作台 DTO 与施工工作台数据，不新增持久化模型。
- 不修改项目详细编辑页、工程量计算规则、财务年度或附件逻辑。

## 验证

- Web 标记测试覆盖两个下拉复选菜单、查看/快捷编辑工程金额控件隔离、里程碑移除和合作单位三类来源。
- JavaScript 测试或源代码测试覆盖选择计数、全选、清空及取消快捷编辑恢复金额口径。
- 项目页面测试确认查询角色仍无快捷编辑入口。
- Release 全量测试和构建通过。
- 仅执行桌面端浏览器验收，确认快捷编辑展开后面板紧凑、菜单可用、无横向溢出和控制台错误；不执行手机端验证。
