## 二、已确认的总体要求

- 新建全新项目，不直接修改 `Reference`。
- Windows Server + IIS + SQL Server。
- 响应式 Web + PWA，电脑和手机共用一套系统。
- 有限离线覆盖阶段成果、工程量、设备进退场、施工/停工日期段、备注和照片；金额、租金和结算仅在线。
- 系统只供同一实际控制人内部使用，支持多个部门、分支机构和自有签约公司。
- 系统级管理员拥有全部系统和业务权限；应用级管理员负责日常业务管理。
- 合作单位统一建档，支持甲方/总包、施工班组、材料供应商、零星供应商多角色。
- 项目可以有多份合同、多个我方签约公司和工程清单。
- 支持暂估量价、结算量价、部分结算和自动金额计算。
- 财务是内部经营台账，不建设完整会计系统。
- 员工分正式员工和劳务员工，工资支持阶段批次、多种工资项目和分批支付。
- 支持简单报销、借支、分红/利息等员工往来。
- 第一阶段不做审批工作流、原生 App、完整甘特图和供应商/班组登录。
- 支持老系统数据平移、主要模块 Excel 导入导出、领导经营汇总台账、数据库和附件备份。
- 支持图表、指标卡、风险提醒、自由筛选字段导出、个人模板和管理员共享模板。
- 自有公司使用管理员可维护的组合分类，支持多账户、证照、公司维度统计和全公司汇总；不建设公司内部往来台账。
- 员工和自有公司均支持零本或多本证书；到期日期为空表示长期有效，按到期前三、二、一个月及过期状态分级提醒。
- 设备逐台建档，覆盖自有/租赁、归属公司、权属转让、项目进退场、施工/停工日期段、单次最终租赁结算、财务关联和非必填维保。
- 关闭公开注册，用户只能由管理员创建。
- 阶段 7～10 在当前分支自动执行，每阶段本地提交，不推送远端。
- 后续日常迭代只执行桌面浏览器验证；手机端不再随每次修改重复验收，待全部功能开发完成后统一执行一次完整手机端总验收。
