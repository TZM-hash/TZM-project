# 员工管理工作台设计

**日期：** 2026-07-23
**状态：** 设计已确认，待实施计划
**参考基准：** 项目管理列表、项目详情总览及工程量、收款、开票、付款、施工详情标签的显示与编辑模式

## 1. 目标

将员工管理重构为与项目管理一致的工作台，统一员工基础资料、应付记录、实际发放追溯、证书和修改动态的展示与编辑方式。

本次设计遵循四个业务边界：

1. 员工管理逐条录入工资、报销、利息分红及其他应付，计算员工应付总额。
2. 工资台账录入实际发放批次，并把批次总额分配到具体员工或班组人员。
3. 中央账本保存一次发放的资金总额；人员分配明细不得重复生成资金流水。
4. 员工详情的付款记录只读追溯工资台账来源，不提供新增、编辑或删除。

## 2. 范围

### 2.1 本次范围

- 员工管理列表页工作台化。
- 员工详情页顶部资料、年度汇总和原位快捷编辑。
- 工资明细、报销明细、利息分红、付款记录、证书管理五个标签。
- 右侧动态、修改记录和风险提示。
- 工资台账、中央账本、个人垫付账户与员工详情的追溯关系。
- 现有员工年度总账、敏感信息权限、证书总览和历史数据兼容。

### 2.2 非目标

- 不在员工付款记录中新增或修改实际付款。
- 不把员工分配金额复制为中央账本的独立资金流水。
- 不建设银行接口、自动付款或外部审批流。
- 不在本次设计中拆分新的独立“员工发放”主模块；工资台账先通过发放类型兼容工资与其他付款。
- 不细分报销类别，报销用途由备注记录。

## 3. 总体页面结构

员工管理采用与项目管理相同的两级结构：

```text
员工管理列表
├─ 全局汇总
├─ 数据工作台
└─ 员工基础资料与年度财务摘要表

员工详情
├─ 员工标题与业务年度
├─ 年度总账汇总
├─ 基础资料快捷编辑
├─ 五个业务标签
│  ├─ 工资明细
│  ├─ 报销明细
│  ├─ 利息分红
│  ├─ 付款记录（只读）
│  └─ 证书管理
└─ 右侧动态与修改记录
```

所有新增区、表格编辑、附件和备注交互复用项目详情的模式：折叠新增区、整表快捷编辑、原单元格输入、统一保存与取消、附件原位上传和备注弹层编辑。

## 4. 员工列表页

### 4.1 标题与操作

取消当前按钮式员工子导航，改为项目列表同款标题区。

- 眉题：人员经营台账。
- 标题：员工管理。
- 说明：统一查看员工主档、当前归属、应付和付款数据。
- 右侧操作：年度总账、证书总览、新增员工。

### 4.2 汇总条

列表顶部显示：

- 员工总数。
- 正式员工。
- 劳务员工。
- 特殊临时人员。
- 在职人数。
- 当前业务年度应付总额。
- 当前业务年度已付款。
- 当前业务年度未付款。

### 4.3 数据工作台

复用项目列表现有数据工作台能力：

- 全字段搜索。
- 员工类型和状态快速筛选。
- 公司、部门、项目、班组、入离职日期、未付款金额范围高级筛选。
- 排序、列管理、行间距、分页和保存视图。
- 列显示配置按用户保存。

### 4.4 默认列

- 序号。
- 员工编号。
- 姓名。
- 员工类型。
- 岗位。
- 电话。
- 当前公司。
- 当前部门。
- 应付合计。
- 已付款。
- 未付款。
- 状态。
- 操作。

### 4.5 可选列

- 入职日期、离职日期。
- 当前项目、当前班组。
- 考勤工资、加班工资、奖金、罚款、其他工资应付。
- 报销金额。
- 利息分红金额。
- 结算进度、备注摘要。
- 有敏感信息权限时可选身份证、银行卡和开户行。

### 4.6 编辑入口

列表不再提供行内“快捷编辑员工”。操作统一为“查看”和“详细编辑”。快捷编辑移动到员工详情基础资料区，避免宽表进入编辑状态后产生横向错位。

## 5. 员工详情顶部

### 5.1 标题区

- 眉题显示员工编号。
- 主标题显示员工姓名。
- 副标题显示员工类型、岗位、当前公司和部门。
- 右侧显示启用状态、详细编辑和复制员工。
- 业务年度选择器控制年度汇总和五个标签中的年度数据。

### 5.2 年度总账汇总

显示：

- 往年结转。
- 工资应付。
- 报销应付。
- 利息分红及其他应付。
- 本年应付合计。
- 已付款。
- 未付款。
- 当前余额和结算进度。

### 5.3 基础资料快捷编辑

基础资料面板分为四组：

1. 人员资料：编号、姓名、类型、状态、岗位、电话、入职日期、离职日期。
2. 敏感资料：身份证、银行卡、开户行。
3. 默认计薪：月薪、日薪、时薪、计件单价和主要计薪方式。
4. 当前归属：公司、部门、项目、班组和归属生效日期。

点击“快捷编辑”后，显示值在原位置切换为输入控件。保存必须携带修改原因和并发标记。复杂、低频或历史归属调整继续通过详细编辑处理。

无敏感信息权限时，身份证和银行卡必须由服务端脱敏，不能只依赖前端隐藏。

## 6. 标签一：工资明细

### 6.1 二级切换

- 全部。
- 考勤工资。
- 加班工资。
- 奖金。
- 罚款。
- 其他。

“其他”包含管理员录入的其他工资应付，以及个人垫付账户自动形成的“个人垫付待归还”。

### 6.2 汇总

显示考勤工资、加班工资、奖金、罚款扣减、其他应付和工资应付合计。罚款以正数录入，在总账中按扣减计算。

### 6.3 表格

统一展示：

- 期间或日期。
- 工资分类。
- 计薪方式。
- 数量与单位。
- 单价。
- 计算金额。
- 调整金额。
- 最终金额。
- 应付公司。
- 项目。
- 班组。
- 附件。
- 备注。

### 6.4 新增与编辑

- 考勤工资支持月薪、计日、计时、计件、固定金额和自定义单位。
- 加班工资支持日期或期间、加班小时或天数、加班单价。
- 奖金填写日期、金额和说明。
- 罚款填写日期、正数金额和原因，保存后按扣减处理。
- 四类手工记录共用应付公司、项目、班组、附件和备注。
- 手工记录支持整表快捷编辑。
- 自动生成的个人垫付待归还记录只读，必须返回原始垫付记录修改。

工资明细不重复显示实际付款。页面可提供“查看工资付款”链接，切换到付款记录并筛选工资来源。

## 7. 标签二：报销明细

### 7.1 汇总

顶部仅显示报销总金额，不显示报销类别、付款状态、已付款或未付款分项。

### 7.2 表格

- 报销日期。
- 报销金额。
- 项目。
- 票据号。
- 附件。
- 备注。

应付公司在新增和编辑表单中必填，用于公司维度总账，但不作为默认表格列显示。

### 7.3 编辑

- 删除报销类别、原始金额和管理员调整字段的页面概念。
- 管理员直接录入和修改最终报销金额。
- 报销用途和分类写入备注。
- 支持折叠新增、整表快捷编辑、附件原位维护和备注编辑。
- 不在本标签显示付款流水或付款状态。

## 8. 标签三：利息分红

### 8.1 汇总与切换

- 顶部仅显示利息分红总金额。
- 二级切换：全部、分红、利息、其他应付。

### 8.2 表格

- 日期。
- 类型。
- 金额。
- 应付公司。
- 项目。
- 附件。
- 备注。

支持折叠新增和整表快捷编辑。该标签只形成应付，不重复显示实际付款。

## 9. 标签四：付款记录

### 9.1 定位

付款记录是只读追溯视图，不提供新增、编辑、删除或快捷编辑。实际付款只能在工资台账的发放批次中维护。

### 9.2 主切换与筛选

- 主切换：全部、工资、其他。
- 工资内部筛选：全部工资、社保工资、民工工资。
- 发放方式筛选：公司账户、私人转账。

“其他”包含报销、利息分红、借支及其他全部非工资付款，且全部计入中央账本。

### 9.3 表格

- 发放日期。
- 工资批次。
- 付款类型。
- 工资性质。
- 发放方式。
- 付款公司。
- 劳务公司。
- 项目。
- 当前员工分配金额。
- 实际收款人。
- 备注。
- 操作。

社保工资不强制劳务公司和项目。民工工资必须关联劳务公司，项目选填；未选择项目时明确显示“未归属项目”。

### 9.4 跳转

操作列显示“查看来源批次”。跳转携带工资批次 ID、人员分配明细 ID 和返回地址，工资台账自动定位并高亮对应人员行。工资台账再提供中央账本总额记录的跳转。

## 10. 标签五：证书管理

标签名称旁显示证书数量，不增加复杂汇总条。

表格展示：

- 证书类型。
- 证书编号。
- 专业、等级或范围。
- 发证机关。
- 签发日期。
- 到期日期。
- 状态。
- 附件。
- 备注。

支持折叠新增、整表快捷编辑、附件上传替换和预览。状态根据有效期自动计算为有效、即将到期、已过期或长期有效。独立证书总览继续保留，用于跨员工筛选。

## 11. 右侧动态与修改记录

历史记录不再占用业务标签，统一放入右侧动态栏：

- 员工主档修改。
- 工资、报销、利息分红及其他应付新增或修改。
- 工资批次付款及分配变化。
- 证书新增、修改、替换和到期提醒。
- 个人垫付形成、归还或冲销。
- 并发冲突、负余额、批次分配差额等风险提示。

移动端将动态栏移动到五个标签之后，不与主内容并排。

## 12. 工资台账与中央账本

### 12.1 工资台账职责

一次发放批次保存：

- 发放类型：工资或其他。
- 发放日期。
- 付款公司。
- 公司资金账户或个人垫付账户。
- 付款方式。
- 实际发放总额。
- 多个员工或班组人员分配明细。
- 工资性质、劳务公司、可选项目和备注。

工资台账名称本期保持不变，但发放类型不得硬编码为只有工资，以便未来平滑扩展为员工发放台账。

### 12.2 中央账本职责

- 一个发放批次对应中央账本一笔总额付款。
- 批次总额必须等于全部员工和班组分配金额之和。
- 人员分配明细不得重复生成中央账本资金流水。
- 员工详情仅显示当前员工的分配金额。
- 私人转账和其他付款同样进入中央账本。

批次、分配明细和中央账本总额记录必须在同一事务中保存或回滚。

## 13. 个人垫付账户

### 13.1 账户模型

个人垫付账户记录：

- 账户名称。
- 实际付款人。
- 可选关联员工。
- 联系方式和收款信息。
- 启用状态。
- 累计垫付、已归还和未归还。

非员工个人允许建档，但只有关联员工的个人账户才能自动形成员工应付。

### 13.2 垫付流程

以张三私人账户垫付为例：

1. 工资台账使用张三个人垫付账户完成一次发放。
2. 中央账本记录本次实际发放总额。
3. 系统自动在张三“工资明细 -> 其他”生成“个人垫付待归还”。
4. 自动应付金额等于张三本次实际垫付金额。
5. 该金额计入张三应付总额和当前余额，但不计入考勤工资、加班工资、奖金或公司的工资成本。
6. 自动应付只读，并保存原始发放批次和个人垫付记录引用。
7. 修改或冲销原始垫付时，在同一事务中同步更新或冲销自动应付。
8. 公司归还张三时，在工资台账以其他付款分配给张三，并从公司资金账户付款。
9. 归还批次进入中央账本，同时冲减张三个人垫付账户未归还金额。

原始工资发放和公司归还个人是两笔不同业务，不覆盖、不复用同一中央账本记录。

## 14. 数据一致性

### 14.1 唯一编辑源

- 员工应付金额：员工管理对应业务标签。
- 实际发放批次和人员分配：工资台账。
- 资金总额：工资台账同步形成的中央账本记录。
- 个人垫付待归还：原始个人垫付自动生成，不允许手工修改。

其他页面只能关联查询或跳转来源，不保存可独立修改的金额副本。

### 14.2 年度口径

- 应付按应付记录日期归入业务年度。
- 付款按发放日期归入业务年度。
- 往年余额按现有年度总账规则结转。
- 跨年度付款仍保留原始应付和付款年度，并由年度总账计算余额。

### 14.3 并发与审计

- 所有手工应付和工资批次保存使用并发标记。
- 快捷编辑必须填写修改原因。
- 审计记录保存修改前后金额、归属、操作人和来源页面。
- 来源记录冲销不得物理删除历史追溯关系。

## 15. 权限

- 员工主档新增和修改：系统管理员、应用管理员。
- 工资、报销、利息分红及其他应付维护：系统管理员、应用管理员、财务。
- 工资批次和人员分配维护：沿用工资管理权限。
- 中央账本总额维护：沿用财务权限，但工资来源记录应跳回工资台账修改。
- 证书维护：沿用员工证书管理权限。
- 查询角色：只读。
- 身份证、银行卡和个人账户收款信息：服务端按敏感信息权限脱敏。

## 16. 校验与错误处理

- 应付金额不得为负数；罚款以正数录入并由系统转换为扣减。
- 手工应付必须选择应付公司。
- 民工工资付款必须选择劳务公司，项目可空。
- 公司账户发放时，资金账户必须属于付款公司。
- 私人转账必须选择有效个人垫付账户。
- 个人垫付自动进入员工应付时，个人账户必须关联员工。
- 核对完成的发放批次必须满足分配合计等于实际总额。
- 保存批次、中央账本总额、个人垫付余额和自动员工应付任一环节失败时整体回滚。
- 并发冲突返回原页面、保持当前标签和编辑状态，并提示刷新后重试。
- 来源记录不存在或无权访问时，付款记录显示明确提示，不提供失效编辑入口。

## 17. 兼容与迁移

- 现有工资、报销、其他应付和领款历史继续保留。
- 现有社保工资、民工工资和其他工资分类映射到新的工资显示分类时必须保留原始来源字段。
- 现有工资批次付款继续作为员工付款记录来源，不复制到新的付款表。
- 无法确定劳务公司或项目的历史民工工资显示“历史未归属”，不在迁移中猜测。
- 现有报销类别字段可以保留在数据库和历史导出中，但新页面不再要求或显示该字段。
- 旧员工详情链接和业务年度参数保持兼容。

## 18. 测试与验收

至少覆盖：

- 员工列表默认列、可选列、筛选、保存视图和权限脱敏。
- 员工详情快捷编辑、取消、并发冲突和修改原因。
- 考勤工资、加班工资、奖金、罚款和其他应付的计算与分类汇总。
- 罚款正数录入后正确扣减工资应付。
- 报销页面只显示确认字段，管理员可直接修改金额。
- 利息、分红和其他应付正确进入员工总账。
- 一个工资批次同时包含多个员工和班组，分配合计等于批次总额。
- 一个批次只形成一笔中央账本总额付款。
- 员工付款记录只显示当前员工分配金额，且无新增和编辑入口。
- 付款记录能够跳转并定位工资批次人员明细。
- 社保工资不要求劳务公司；民工工资要求劳务公司但允许项目为空。
- 其他付款包含报销、利息分红、借支和其他非工资付款。
- 私人转账进入中央账本并增加个人垫付余额。
- 关联员工的个人垫付自动生成“个人垫付待归还”，计入员工应付但不计工资成本。
- 修改、冲销个人垫付时自动应付同步变化。
- 公司归还个人后，个人垫付余额和员工付款记录同步更新。
- 非员工个人垫付不错误生成员工应付。
- 证书状态、附件和跨员工证书总览保持正确。
- 移动端标签、表格、快捷编辑和动态栏不重叠。

## 19. 验收标准

完成后应满足：

1. 员工列表和详情在视觉、信息密度、编辑方式和错误反馈上与项目管理一致。
2. 员工应付、工资批次分配和中央账本总额之间不存在可独立修改的金额副本。
3. 所有付款均可从员工详情追溯到工资批次，再追溯到中央账本总额。
4. 个人垫付能够自动形成对应员工待归还应付，并在公司归还后正确结清。
5. 历史数据、业务年度结转、敏感信息权限和审计关系保持可用。
