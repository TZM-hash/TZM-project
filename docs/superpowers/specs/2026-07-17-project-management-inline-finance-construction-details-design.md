# 项目管理原位编辑与施工详情设计

## 1. 目标与范围

本轮仅调整项目管理大类及其同步所必需的底层服务，不调整员工、合作单位、设备、公司等其他页面的界面。

目标：

- 删除项目总览中的“上级项目”“分支机构”。
- 增加实际开工日期、实际完工日期、收款比例、开票比例、付款比例。
- 项目总览、工程量、收款、开票、付款全部采用显示位置原位编辑。
- 新增“施工详情”标签，统一管理设备和施工班组的多次进退场及项目流转。
- 项目页面直接新建设备或施工班组时创建正式主档，并自动同步其他页面。
- 项目管理页面可直接修改正式财务数据，同步账户流水、汇总和其他页面，并保留完整修改日志。

## 2. 项目总览布局

项目总览保留四列网格，字段顺序为：

1. 项目编号、项目名称、项目阶段、合作方式。
2. 总包单位、总包联系人、联系电话、项目负责人。
3. 部门、签约公司、归档状态、实际开工日期。
4. 实际完工日期、收款比例、开票比例、付款比例。
5. 合同金额、暂估金额、结算金额、应付/已付/未付。

比例统一读取项目详细页的 `FinanceSummary`：

- 收款比例 = 已收款 / 应收款。
- 开票比例 = 已开票 / 应收款。
- 付款比例 = 已付款 / 应付款。
- 分母为 0 时显示 0%；文字显示真实百分比，进度条视觉宽度最多为 100%。

实际完工日期允许为空，但不得早于实际开工日期。

## 3. 原位快捷编辑规范

- 点击标签页或总览区域的“快捷编辑”后，原文字所在位置切换为输入框、日期框或中文下拉框。
- 控件预填当前正式值；用户可删除原值、覆盖新值，空值直接显示空控件。
- 不弹窗，不在其他位置展开独立大表单。
- 每条明细独立保存、独立取消；保存失败时保留用户输入并在当前行显示中文错误。
- 一个区域或一行进入编辑，不影响其他区域或其他行。
- 比例、总天数、施工天数等派生值只读，由系统实时计算。

## 4. 财务正式数据修改

项目管理页面拥有最高编辑优先级：

- 系统管理员、应用管理员、项目经理可修改项目资料、工程量、全部财务明细和施工详情。
- 财务人员可修改项目财务明细，不能修改项目基础资料、工程量和施工详情。
- 设备管理员可查看施工详情设备记录，不能从项目页修改。
- 查询用户和现场人员只读。

项目页直接修改正式应收、收款、发票、应付和付款记录：

- 修改收款或付款的账户、日期、金额时，同一数据库事务内更新对应 `AccountTransaction`。
- 修改发票金额时保留原关联对象，并按新含税金额同比调整关联分配，保证分配合计不超过新金额。
- 修改项目、合同、签约公司、往来单位等归属字段后，所有页面读取同一正式记录并立即同步。
- 每次修改保存修改原因、操作人、操作时间、修改前 JSON 和修改后 JSON。
- 任一步失败时整个事务回滚；使用并发版本阻止覆盖他人新修改。
- 不提供物理删除；已有作废、冲销业务继续保留。

## 5. 施工详情

“施工详情”位于“付款明细”右侧。统一表格列：

- 类型（设备/施工班组）
- 设备或班组
- 从哪个项目调入
- 进场日期
- 退场日期
- 总天数
- 停工天数
- 施工天数
- 退场到哪个项目
- 备注
- 操作

同一设备或班组在同一项目允许多次进退场，每次形成独立记录。

计算规则：

- 已退场：总天数 = 退场日期 - 进场日期 + 1。
- 未退场：总天数 = 今天 - 进场日期 + 1。
- 施工天数 = 总天数 - 停工天数。
- 停工天数必须在 0 至总天数之间。

项目流转规则：

- 调入项目、调出项目均可留空或手动选择。
- 自动连接上个项目时，匹配同一设备或班组最近的上一条记录。
- 选择调出项目或自动连接下个项目时，在目标项目自动创建待完善记录，并建立双向上一条/下一条关系。
- 修改流转关系时解除旧连接、建立新连接；禁止连接自身、形成循环或重复生成同一待完善记录。

## 6. 数据结构

`Project` 新增：

- `ActualStartDate DateOnly?`
- `ActualCompletionDate DateOnly?`

新增 `ProjectConstructionRecord`：

- `Id`
- `ProjectId`
- `RecordType`（Equipment / ConstructionCrew）
- `EquipmentId?`
- `CrewBusinessPartnerId?`
- `TransferFromProjectId?`
- `TransferToProjectId?`
- `PreviousRecordId?`
- `NextRecordId?`
- `EntryDate?`
- `ExitDate?`
- `StopDays`
- `Notes?`
- `IsDraft`
- `CreatedAt`
- `UpdatedAt`
- `ConcurrencyStamp`

必须且只能选择设备或施工班组之一。班组必须具有 `ConstructionCrew` 角色。

## 7. 项目内新建正式主档

施工详情选择器提供“新建设备”和“新建施工班组”：

- 新建设备必填设备编号、设备名称；型号、分类、权属等可选。
- 新建班组必填单位编号、班组名称，自动添加施工班组角色；联系人、电话可选。
- 创建调用现有正式主档服务，成功后自动选中新记录并建立施工记录。
- 其他页面自动读取到新主档；不修改其他模块页面布局。

## 8. 异常处理与验收

- 退场日期不得早于进场日期。
- 完工日期不得早于开工日期。
- 财务金额保持现有正数与关联校验。
- 流转连接必须无环且成对更新。
- 数据写入、账户流水、关联分配和审计必须在同一事务完成。
- 自动化测试覆盖日期、比例、权限、原位编辑、财务同步、账户流水、审计、多次进退场、自动流转、循环保护和主档同步。
- 运行完整 Release 测试，并执行 1440px 桌面浏览器验收。
- 不做手机端验证，不执行 Git 暂存、提交或推送。
