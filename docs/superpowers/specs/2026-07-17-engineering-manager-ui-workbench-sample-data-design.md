# 工程项目经营管理系统 UI、数据工作台与完整测试数据设计

版本：v1.0

日期：2026-07-17

状态：已确认，待实施计划

## 1. 背景与目标

当前系统已经完成阶段 0～10 的业务功能，但前端只具备基础响应式外壳和轻量统计图，视觉完成度、用户操作效率和完整数据展示均未达到最初约定的 `Reference` 参考项目水平。现有 Development 样例数据也仅包含单家公司、单个项目、单名员工、单台设备和单份合同，无法展示真实使用状态。

本次作为“阶段 10 后续体验优化迭代”，目标是：

1. 删除并重建明确标识的 `EngineeringManager_Test` 测试数据库，生成中等真实规模的完整关联样例。
2. 以 `Reference` 项目的成熟设计系统为基准，重做应用外壳、页面风格、图形化展示、显示主题和动效。
3. 增加系统级显示与交互设置页面。
4. 将列管理、行间距、筛选、排序、分页、个人保存视图和导出联动等数据工作台能力覆盖到主要列表页。
5. 保持现有 ASP.NET Core Razor Pages、业务模型、权限边界、有限离线范围和简体中文界面。

## 2. 已确认决策

| 项目 | 决策 |
| --- | --- |
| 测试数据库 | 删除并重建 `EngineeringManager_Test`，禁止触碰生产数据库 |
| 测试数据规模 | 中等真实规模，覆盖近 12 个月完整业务链 |
| UI 基准 | 完整采用 `Reference` 的成熟界面体系并按本系统业务适配 |
| 实施路线 | 提炼复用设计系统和用户操作组件，不整体复制无关代码 |
| 设置范围 | 系统级全局设置，由系统管理员统一管理 |
| 图表实现 | 本地 SVG、Canvas 和语义化 HTML，不连接外部 CDN |
| 显示语言 | 固定简体中文，不移植繁简转换 |
| 用户偏好 | 主题全局统一；个人保存视图可跨设备；临时布局保存在浏览器 |
| Git | 稳定交付点本地提交，不主动推送远端 |

## 3. 范围边界

### 3.1 本次包含

- Development 测试库安全重建和完整样例播种。
- 共享应用外壳、导航、顶部栏、图标、设计令牌和响应式结构。
- 全局主题、字体、动效、特效等级和表格密度设置。
- 首页及项目、财务、员工、工资、自有公司、设备等模块图形化摘要。
- 主要列表的数据工作台能力和个人保存视图。
- 登录、空状态、错误状态、无权限状态和加载反馈统一设计。
- 桌面、平板、手机、键盘操作和减少动态效果适配。
- 自动化测试、浏览器验收、进度文档和本地 Git 提交。

### 3.2 本次不包含

- 修改现有业务核算口径、审批规则、角色定义或有限离线边界。
- 引入 SPA 前端框架或外部 CDN。
- 把主题改为每个用户单独配置。
- 移植 `Reference` 中的繁简转换、甘特档案日期和无关业务页面。
- 无条件增加批量删除、批量修改合同金额等高风险操作。
- 操作生产数据库、生产服务器、正式附件或远端 Git 推送。

## 4. 交付顺序与架构

实施按以下连续交付点推进：

1. 安全重建测试数据库并生成完整演示数据。
2. 建立测试数据关系与金额一致性验证。
3. 移植并精简 `Reference` 的四层 CSS、模块化 JavaScript、SVG 图标和应用外壳。
4. 建立系统设置存储、权限、审计、缓存和实时预览页面。
5. 建立通用数据工作台组件和个人保存视图。
6. 重做首页经营驾驶舱和各业务模块图形化摘要。
7. 逐页统一列表、详情、表单、登录和管理中心。
8. 完成全量测试、浏览器验收、开发进度更新和本地提交。

继续采用模块化单体和服务器端渲染。共享视觉行为放入独立 CSS/JavaScript 层，业务聚合放入应用服务，页面模型仅负责权限范围、查询参数和视图模型组装。

## 5. 视觉系统与应用外壳

### 5.1 视觉方向

主视觉采用 `Reference` 的蓝青科技商务风：

- 浅灰蓝应用背景和白色主工作区。
- 蓝色主操作、青色强调以及绿/橙/红状态色。
- 克制的阴影、描边、高光和毛玻璃层次。
- 密集但保持呼吸感的业务表格与表单。
- 卡片圆角保持中等，不使用夸张营销式大圆角。
- 不使用外部字体、图片或图表服务。

### 5.2 应用外壳

- 左侧可收缩导航按“经营总览、项目经营、人员与合作方、公司与设备、数据与系统”分组。
- 顶部栏显示当前页面、快捷搜索、提醒、网络/同步状态和用户菜单。
- 当前导航根据路由自动激活，不再固定高亮首页。
- 侧栏收缩状态保存到当前浏览器。
- 桌面使用固定侧栏；手机和平板窄屏使用抽屉导航。
- 导航和操作图标使用本地 SVG，不使用 Unicode 字符图标。

### 5.3 页面模板

列表页统一为：

```text
页面标题与主操作 → KPI 摘要 → 快速/高级筛选 → 图形摘要 → 数据工作台 → 分页/导出
```

详情页统一为：

```text
核心状态与操作 → 金额/进度图表 → 分页标签 → 业务明细 → 附件 → 修改日志
```

表单页按业务分区，包含必填标识、字段说明、验证反馈和稳定的保存/返回操作区。

## 6. 图形化展示

### 6.1 首页经营驾驶舱

- 在管项目、当前工程金额、未收款、未付款、未开票、未发工资、设备成本和未处理提醒 KPI。
- 近 12 个月收入、支出、收款和付款趋势。
- 应收/已收/未收、应付/已付/未付、应开票/已开票/未开票对比。
- 项目阶段分布和高风险项目排行。
- 工资、设备成本和本设备离线同步摘要。

### 6.2 业务模块

- 项目：合同额、暂估额、结算额，工程量完成度，收款/开票进度，多签约公司拆分。
- 财务：月度收支、应收应付、账户余额和资金流向摘要。
- 员工与工资：正式/劳务员工分布、阶段工资趋势、未发工资、报销和其他员工往来。
- 自有公司：各公司合同额、收款、开票、未收款和项目数量。
- 设备：自有/租赁分布、项目使用天数、施工/停工比例、租金和内部参考成本。

图表和 KPI 必须与当前筛选条件共用同一查询范围。图表点击只执行明确的筛选或导航，不直接产生业务写入。

### 6.3 渲染与可访问性

- 使用本地 SVG、Canvas 和语义化 HTML，不连接外部 CDN。
- 图表同时提供文字、数值和无数据状态，不能只依赖颜色或悬停。
- 默认启用数字滚动和图表展开；轻特效或减少动态效果模式关闭非必要动画。
- 手机端优先显示风险和核心 KPI，再显示图表和明细入口。

## 7. 系统显示与交互设置

新增“系统管理 → 显示与交互设置”。

### 7.1 设置项

- 显示主题：默认科技商务、清透毛玻璃。
- 动效风格：科技商务动效、Apple 弹性动效。
- 特效等级：轻度、中度、高度，默认中度。
- 全局字体：系统默认、微软雅黑、微软正黑、宋体/明体、楷体。
- 表格密度：紧凑、标准、宽松，默认标准。

设置页面提供卡片式实时预览，保存后立即全站生效。系统固定使用简体中文；不提供繁简转换或无关的甘特设置。

### 7.2 权限、存储和审计

- 系统级管理员可以修改。
- 应用级管理员可以查看，但不能修改。
- 设置使用独立轻量键值表保存，通过测试库 Migration 建立。
- 每次保存记录修改人、时间、修改前值和修改后值。
- 读取使用短期缓存，保存后立即失效。
- 操作系统或浏览器要求减少动态效果时，强制降低动画，不被全局高特效覆盖。

## 8. 数据工作台

主要列表页共享以下能力：

### 8.1 列与密度

- 显示或隐藏可配置列。
- 拖动调整可配置列顺序。
- 恢复系统默认布局。
- 关键名称列和操作列允许固定。
- 行间距支持紧凑、标准、宽松。

### 8.2 搜索、筛选和排序

- 常用字段模糊搜索。
- 高级筛选抽屉支持状态多选、公司/部门/负责人、日期范围、金额范围和业务布尔条件。
- 当前条件显示为可单独移除的筛选标签，并支持一键清除。
- 表头支持升序、降序和取消排序。
- 查询参数写入 URL，刷新和复制链接后保持筛选结果。

### 8.3 分页、保存视图和导出

- 每页 20、50、100 条。
- 个人保存视图包含筛选、排序、可见列、列顺序、行距和每页数量。
- 用户可命名视图并指定自己的默认视图。
- 系统提供施工中、未收款、待开票、未结算设备、未发工资等预设视图。
- 未保存的临时布局保存在浏览器；个人保存视图写入数据库并跨设备使用。
- Excel 默认导出当前筛选结果和可见列，同时保留临时调整导出字段的入口。

### 8.4 操作安全

- 批量选择用于批量导出和现有安全批处理。
- 不新增通用批量删除或关键金额批量修改。
- 所有查询继续执行当前用户角色和项目数据范围过滤。
- 手机端默认保留关键字段，其余字段通过列管理或详情查看。

## 9. 完整测试数据

### 9.1 数据规模

- 5 家自有公司，覆盖一般纳税人、小规模纳税人、个体户等组合分类、不同法人、账户和证照状态。
- 15 个项目，覆盖待进场、施工中、阶段暂停、完工未结算和已结算。
- 18～20 份合同；多数项目一家签约，少数项目由 2～3 家自有公司拆分记录。
- 暂估及最终工程量、单价和结算数据。
- 近 12 个月应收、收款、开票、应付、付款、账户收支和部分冲销记录。
- 30 名正式/劳务员工，覆盖月薪、阶段工资、项目工资、借支、报销、奖金、利息和分红。
- 12 个合作单位/施工班组，覆盖总包方、供应商、班组和设备出租方等复合角色。
- 15 台自有/租赁设备，分属不同自有公司，并包含进退场、施工/停工日期段、租金和结算。
- 阶段成果、工程量汇总、备注及少量明确标识为测试资料的附件/现场图片占位文件。
- 设备租赁到期、维保到期、未收款、未开票和同步异常等提醒。

### 9.2 测试账号

生成系统管理员、应用管理员、财务人员、项目负责人和现场人员账号。密码保持本地测试所需的适中复杂度，写入 Git 忽略的 `App_Data/local-test-credentials.txt`，不写入源码、提交记录或聊天内容。

### 9.3 一致性

- 未收款等于应收减已收。
- 未付款等于应付减已付。
- 未开票等于应开票减已开票。
- 暂估和结算金额与相应工程量及单价一致。
- 设备使用天数等于进退场区间内施工和停工日期段的有效汇总。
- 公司、项目、合同、员工、设备和财务数据的外键关系完整。

样例使用相对当前日期生成近 12 个月数据，并使用稳定业务编号。播种器保持幂等，意外重复执行不会重复插入。

## 10. 测试数据库安全

删除和重建前必须同时满足：

1. 环境名严格为 `Development`。
2. 数据库名严格以 `_Test` 结尾。
3. 连接目标与项目开发配置一致。

任一条件不满足时立即停止。自动化不得读取、删除、迁移或写入生产数据库。测试库重建失败时停止后续 UI 验收，不把部分数据报告为完成。

## 11. 异常处理

- 样例数据分批事务写入；核心关系失败时回滚当前批次并报告具体对象。
- 个人保存视图遇到已删除字段时忽略失效字段并恢复可用默认布局。
- 图表在空数据、零金额、无权限和查询失败时显示明确状态。
- 设置保存失败时保留输入并显示原因；成功后立即清理缓存。
- 筛选、导出、保存视图和长时间处理提供加载、成功和错误反馈。
- 关键设置、保存视图和业务写入继续记录审计日志。
