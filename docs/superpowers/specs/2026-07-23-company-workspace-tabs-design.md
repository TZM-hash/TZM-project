# 自有公司工作台：下拉切换与标签页设计

## 1. 目标与范围

将「自有公司」从「列表 + 详情散点」升级为统一经营工作台：

- 顶部公司范围下拉可在「全部公司」与各具体公司之间切换。
- 选择具体公司后，以标签页组织经营概览、基本信息、证书、账户、项目与合同、收付款与发票。
- 档案类信息在公司页内原位维护；业务明细只读展示，并通过超链接进入原业务模块查看/编辑。

### 1.1 本轮包含

- `/Companies` 全部公司总览页改造（指标卡收敛、统一下拉框架）。
- `/Companies/Details/{id}` 具体公司工作台（标签 + URL `tab`）。
- 基本信息、证书、账户的标签内维护（含账户启用/停用强视觉）。
- 项目与合同、收付款与发票的只读列表与业务深链。
- 按 tab 加载与权限边界。

### 1.2 本轮不包含

- 在公司页内嵌完整项目/财务编辑表单。
- 将总览与详情强行合并为单一超重 URL 页面。
- 审批流、物理删除公司或账户。
- 新建内部公司往来账本专页。
- 手机端完整视觉重做（沿用现有响应式与表格横滑）。
- 前端 SPA 局部异步切换公司（采用服务端分路由方案）。

## 2. 已确认决策

| 决策点 | 结论 |
| --- | --- |
| 默认落地 | 保留「全部公司」总览；下拉首项为「全部公司」 |
| 具体公司形态 | 完整经营工作台（6 个标签） |
| 架构方案 | 统一外观、分离路由：`/Companies` + `/Companies/Details/{id}?tab=` |
| 档案编辑 | 当前标签内原位/就地编辑，保存后停留原标签 |
| 业务编辑 | 公司页只读；点击名称/单号跳转原业务模块 |
| 全部公司指标卡 | 仅「公司数量」「未收款」；不显示合同金额、账户余额 |
| 账户状态 | 每账户可启用/停用；停用须有明确视觉；文案不用「删除/恢复」 |

## 3. 信息架构与路由

### 3.1 路由

| 场景 | 路由 | 说明 |
| --- | --- | --- |
| 全部公司总览 | `/Companies` | 菜单「自有公司」默认页 |
| 具体公司工作台 | `/Companies/Details/{id}?tab={tab}` | `tab` 缺省或非法时视为 `overview` |
| 新增公司 | `/Companies/Edit` | 独立页，不并入标签 |
| 详细档案编辑（保留） | `/Companies/Edit/{id}` | 详情「进入详细编辑」入口；日常以标签原位编辑为主 |
| 复制公司 | `/Companies/Edit?copyFrom={id}` | 沿用现有能力 |

### 3.2 标签定义（固定顺序）

| 顺序 | `tab` 值 | 中文名 | 类型 |
| --- | --- | --- | --- |
| 1 | `overview` | 经营概览 | 只读汇总 |
| 2 | `profile` | 基本信息 | 档案，可原位编辑 |
| 3 | `certificates` | 证书信息 | 档案，可就地维护 |
| 4 | `accounts` | 账户信息 | 档案，可就地维护 + 启停 |
| 5 | `projects` | 项目与合同 | 业务只读 + 超链接 |
| 6 | `finance` | 收付款与发票 | 业务只读 + 超链接 |

### 3.3 公司范围下拉

- 选项 1：`全部公司` → `/Companies`
- 选项 2..N：当前 `CompanyActor` 权限范围内的自有公司（显示公司名称）
- 在详情页切换公司：跳转到目标公司 `/Companies/Details/{newId}?tab={当前tab}`；若目标页不支持该 tab 语义则回 `overview`（本轮 6 个 tab 对所有公司一致，故保持当前 tab）
- 在总览页选择具体公司：进入 `?tab=overview`

### 3.4 URL 与浏览器行为

- 切换标签只更新 `tab` 查询参数。
- 刷新页面保持公司与标签。
- 浏览器后退可回到上一标签或上一公司。
- 无效 `tab`：静默回退 `overview`，不报错。

## 4. 页面骨架

### 4.1 共用顶部框架

全部公司页与具体公司页共用视觉骨架：

1. 眉标：`公司维度经营统计`
2. 标题：`自有公司`
3. 说明：分别查看各公司合同、收付款、发票、工资、账户和项目数据
4. 公司范围下拉（见 3.3）
5. 主操作：有管理权限时显示「新增公司」；具体公司页另显示「复制公司」

### 4.2 全部公司页 `/Companies`

1. **指标卡（仅 2 张）**
   - 公司数量
   - 未收款 = 应收 − 已收
2. **金额对比面板**（保留）
   - 应收 / 已收
   - 应付 / 已付
   - 销项 / 进项发票
   - 工资 / 设备成本
3. **公司列表**
   - 搜索：公司全部信息关键词
   - 列：编码、公司、分类、法人/经营者、账户（启用数/总数）、备注摘要、状态、操作
   - 公司名 / 账户数 / 查看 → 该公司详情
   - 编辑 → 详细编辑页或详情 `profile`（实现时优先详情 `profile` 快捷编辑；详细编辑入口仍保留）

### 4.3 具体公司页 `/Companies/Details/{id}`

1. **公司标题条**  
   编码 · 分类 · 全称 · 简称 · 启用/停用
2. **顶部经营速览条**（所有标签始终可见）
   - 应收 / 已收
   - 应付 / 已付
   - 未收款
   - 未付款  
   不显示合同金额、账户余额。
3. **标签导航** + **当前标签内容区**

## 5. 各标签规格

### 5.1 经营概览 `overview`（只读）

**正文：**

1. 经营对比（当前公司口径，结构同全部公司对比面板）
2. 结构摘要卡
   - 关联项目数
   - 合同份数
   - 启用账户数 / 账户总数
   - 有效证书数 / 证书总数；过期数单独红色计数
3. 近期动态（最多 10 条）
   - 类型：项目 / 合同 / 收款 / 付款 / 发票
   - 摘要、金额、日期
   - 整行可点，跳转对应业务页

无编辑按钮。

### 5.2 基本信息 `profile`

**查看字段：**

| 分组 | 字段 |
| --- | --- |
| 身份 | 公司编码、公司名称、简称、组合分类、状态（启用/停用） |
| 主体 | 法人/经营者、统一社会信用代码、联系电话、开票抬头 |
| 地址 | 注册地址、经营地址 |
| 备注 | 备注 |

**编辑规则：**

- 管理权限显示「快捷编辑公司」。
- 点击后字段原位切换为输入框/下拉（沿用现有 inline-edit 模式）。
- 必填：编码、名称、简称、组合分类、修改原因。
- 公司启用/停用可在本标签修改（管理员）。
- 保存成功：停留 `profile`，成功提示，刷新并发戳。
- 取消：恢复查看态。
- 另保留「进入详细编辑」→ `/Companies/Edit/{id}`。

### 5.3 证书信息 `certificates`

**列：** 资料类型、资料编号、签发日期、有效期、到期状态、备注、操作。

**规则：**

- 有效期为空 = 长期有效。
- 到期状态：正常 / 30 天内 / 60 天内 / 90 天内 / 已过期（与现有证书提醒逻辑一致展示）。
- 行内编辑；底部新增证照表单（管理权限）。
- 徽章：已过期红、临期橙、正常灰绿。
- 本轮不新增复杂附件重传；已有附件只读链接可保留。

### 5.4 账户信息 `accounts`

**表头：** `当前启用 X 个，共 Y 个。` 停用账户仍列出。

**列：** 账户名称、账号、开户行、账户类型（银行/现金/其他）、期初余额、默认用途（收款/付款/开票）、状态、账户备注、操作。

**行内维护：** 编辑 / 保存 / 取消；新增面板在标签底部。

**启用 / 停用（强制）：**

1. 每个账户可切换启用、停用。
2. 按钮文案仅为「停用」「启用」，禁止「删除」「恢复」作为主文案。
3. 停用视觉（须同时具备）：
   - 整行浅灰/降对比
   - 状态徽章：绿「启用」/ 灰「已停用」
   - 账户名称后灰色标签 `[已停用]`
4. 停用时自动清除默认收款/付款/开票标记。
5. 确认文案：
   - 停用：`确认停用该账户？停用后不能作为新的默认收付款/开票账户，历史记录保留。`
   - 启用：`确认重新启用该账户？`
6. 排序：启用在前，停用在后；同组按账户名称。
7. 默认用途唯一性等继续遵循现有 `CompanyAccountRules`。
8. 历史收付款若引用停用账户：显示名称并带「已停用」后缀，金额只读。
9. 业务模块新建收付款的账户候选默认仅启用账户；若现有逻辑未限制，实现计划中单列联动项。

### 5.5 项目与合同 `projects`（只读 + 超链接）

**筛选：** 阶段（可选）、关键词（项目编号/名称）；仅本公司签约/分配相关。

**项目列：** 项目编号（链）、项目名称（链）、阶段、合同金额（本公司份额）、应收/已收、应付/已付、打开项目。

**合同：** 子表或展开行；合同编号/名称链接到项目详情合同区域；显示本公司份额金额或比例、合同总额、状态（若有）。

**编辑边界：** 无保存；点击进入 `/Projects/Details/{id}` 等业务页编辑。

### 5.6 收付款与发票 `finance`（只读 + 超链接）

同一标签内上下三块（实现简单，不做二级路由）：

1. 收款明细  
2. 付款明细  
3. 发票明细（销项/进项用方向列区分）

**收款/付款列：** 日期、项目（链）、单据/摘要（链到可编辑业务入口）、账户（停用加后缀）、金额、查看。

**发票列：** 方向、发票号（链）、项目（链）、开票公司、含税金额、日期。

**编辑边界：** 公司页不改金额/账户/作废；全部经超链接进入原业务模块。跳转尽量带 `LegalEntityId={当前公司}` 或等价筛选。

若某类业务暂无稳定单条深链：允许退化为带筛选的列表/项目详情定位；实现计划标注「可深链 / 仅列表」。

## 6. 权限

沿用 `CompanyActor`：

| 动作 | 系统/应用管理员 | 有公司查看范围的业务角色 | 无该公司权限 |
| --- | --- | --- | --- |
| 全部公司总览 | 是 | 仅可见范围内公司数据 | 否 |
| 进入具体公司与 6 标签 | 是 | 仅授权公司 | 否 |
| 新增/复制公司 | 是 | 否 | 否 |
| 编辑基本信息/证书/账户/启停 | 是 | 否 | 否 |
| 点击业务超链接 | 是 | 是（落地页再鉴权） | 否 |

无权限改 URL 访问：不展示数据；`Forbid` 或中文「未找到该公司或无权查看。」，不泄露存在性细节。

业务页编辑权不因从公司页跳入而放宽。

## 7. 状态与反馈

### 7.1 空状态

| 位置 | 文案 |
| --- | --- |
| 无公司 | 当前权限范围内没有自有公司。 |
| 无动态 | 暂无近期项目、收付款或发票动态。 |
| 无证书 | 还没有维护公司证照。 |
| 无账户 | 还没有维护公司账户。 |
| 无项目合同 | 本公司暂无关联项目或合同。 |
| 无收款/付款/发票 | 本公司暂无收款/付款/发票记录。 |

空状态不隐藏下拉、速览条与标签导航。管理员在档案空态旁显示新增入口。

### 7.2 错误与成功

- 校验失败：停留当前标签与编辑态，中文错误。
- 账户行失败：仅该行保持编辑态。
- 启停失败：不乐观更新行状态。
- 并发冲突：`数据已被他人更新，请刷新后重试。`
- 业务标签查询失败：标签内错误面板 + 重试。
- 保存成功：同 tab 停留 + 简短成功提示。

## 8. 数据加载

1. `/Companies`：列表 + 全公司 Dashboard + 下拉数据；不拉各公司明细流水。  
2. Details 公共：标题、下拉、速览 4 指标、`CanManage`。  
3. 按 tab 加载：
   - `overview`：对比、结构摘要、动态 Top 10
   - `profile`：档案；管理端含分类与快捷编辑模型
   - `certificates`：证书列表
   - `accounts`：账户全量（含停用）
   - `projects`：关联项目/合同（默认最近/上限 50，可筛）
   - `finance`：收/付/票各默认最近 50；更多进业务模块
4. 写操作后：重载当前公司当前 tab。  
5. 不在本轮引入新图表库。

### 8.1 预计服务扩展（只读）

在现有 `ICompanyManagementService`（列表、详情、Dashboard、账户/证书保存）之上新增或扩展：

- 公司关联项目与合同列表（含本公司份额）
- 公司维度收款、付款、发票最近列表
- 经营概览结构摘要与近期动态

不在公司模块复制财务写入逻辑。

## 9. 业务超链接约定

| 来源 | 目标 |
| --- | --- |
| 项目编号/名称 | `/Projects/Details/{projectId}` |
| 合同 | 项目详情合同区域 |
| 收款/付款/发票 | 优先项目详情对应分区；否则经营台账/财务页并带公司或项目上下文 |
| 返回 | 浏览器后退；或 `returnUrl` 回 `/Companies/Details/{id}?tab=projects|finance`（实现可选，推荐） |

## 10. 非目标（重申）

- 公司页内完整业务编辑
- 单页合并全部查询到 `IndexModel`
- 审批、物理删除、内部往来专页
- 手机端专项重设计
- SPA 无刷新公司切换

## 11. 验收标准

### 导航与骨架

1. 菜单进入默认为全部公司总览；下拉首项为「全部公司」。  
2. 选择具体公司进入详情，URL 含 `tab`，默认 `overview`。  
3. 切换 6 标签时 `tab` 同步；刷新不丢标签。  
4. 总览与详情下拉选项一致（权限过滤后）。

### 指标

5. 全部公司指标卡只有公司数量、未收款。  
6. 不出现合同金额、账户余额指标卡。  
7. 详情速览为应收/已收、应付/已付、未收款、未付款。

### 档案

8. 基本信息可原位编辑保存；失败保留输入。  
9. 证书可新增与行内维护。  
10. 账户可新增与行内编辑。

### 账户启停

11. 每账户可启停且有确认框。  
12. 停用具备行样式 + 徽章 + 名称标记。  
13. 停用后历史保留且列表仍可见。  
14. 主操作文案为启用/停用，不以删除账户作为主文案。

### 业务只读与跳转

15. 项目与合同、收付款与发票无行内保存。  
16. 项目名/关键摘要/发票号可进入对应业务页。  
17. 只读用户无档案编辑与启停，仍可切换标签并打开业务链接。

### 异常

18. 无效 `tab` 回退 `overview`。  
19. 无权限公司不可通过改 URL 查看。  
20. 并发冲突有中文提示。

## 12. 实现影响面（供后续计划拆分）

- Web：`Pages/Companies/Index.*`、`Details.*`；可能抽取共用公司下拉 partial。  
- Application/Infrastructure：公司只读聚合查询扩展；Dashboard 指标卡字段使用方式调整（未收款展示）。  
- 账户启停：复用/调整 `OnPostAccountStatus`，更名文案与停用清默认规则。  
- 测试：`CompanyPageTests`、`CompanyManagementServiceTests` 增补标签路由、启停视觉相关断言（服务层规则）与权限。  
- 样式：停用行、状态徽章、标签导航，优先复用现有 button/panel/table 体系。
